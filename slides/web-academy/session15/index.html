왜 GitHub를
써야 할까?

Git은 내 PC 안에서만 돌아간다. 하지만 개발은 대부분 협업이다.
팀원과 코드를 공유하고, 리뷰하고, 합치는 공간이 필요하다.

그래서 Git 저장소를 인터넷에 올려둔 플랫폼 — GitHub.
오늘날 1억 명 이상의 개발자가 사용하는 표준.

04 / 15

Git vs GitHub

헷갈리기 쉽지만 완전히 다른 것이다.

🔧 Git
버전 관리 도구(소프트웨어)
내 PC에서 로컬로 동작
오픈소스, 무료
2005년 출시
☁️ GitHub
Git 저장소를 호스팅하는 서비스
웹에서 접근, 협업·리뷰 기능
Microsoft 소유 (2018년 인수)
GitLab, Bitbucket 같은 대안도 존재
05 / 15
🔀

Pull Request —
더 깊게 이해하기

PR은 단순한 "코드 합치기"가 아니라, 팀이 코드 품질을 지키는 핵심 프로세스다.

1
브랜치 생성 & 코드 작성
feature/login 같은 브랜치를 만들어 기능을 개발하고 여러 번 커밋한다.
2
원격 push & PR 생성
GitHub에 브랜치를 올리고 "이 변경을 main에 합쳐주세요"라는 리뷰 요청을 만든다.
3
코드 리뷰
팀원이 코드를 읽고 댓글을 달거나 변경을 요청한다. 이 과정이 품질을 높인다.
4
머지(Merge)
리뷰가 승인되면 main에 병합. 내 코드가 공식적으로 프로젝트의 일부가 된다.

☁️

핵심 명령어 4종

⬆️
push
내 로컬 커밋을 원격 저장소에 올리기. 내가 한 작업을 공유.
⬇️
pull
원격의 변경사항을 내 로컬로 가져오기. 팀원의 작업을 반영.
📥
clone
원격 저장소를 통째로 내 PC에 다운로드. 처음 한 번.
🍴
fork
남의 저장소를 내 계정으로 복사. 오픈소스 기여의 시작점.

⚡

GitHub Actions —
실제로는 어떻게?

코드를 push하는 순간, 자동으로 테스트하고 배포까지 완료된다.

📝 코드 push
→
🤖 Actions 실행
→
✅ 테스트 통과
→
🚀 자동 배포

핵심: 사람이 수동으로 배포할 필요 없이 코드 변경만으로 전체 CI/CD 파이프라인이 동작한다

08 / 15

Pull Request — 협업의 꽃

팀원이 리뷰하고 합치는 5단계 흐름.

1
브랜치 생성 & 작업
feature/login 같은 브랜치에서 기능을 개발하고 커밋.
2
원격에 push
내 브랜치를 GitHub 원격 저장소에 업로드한다.
3
Pull Request 생성
"내 브랜치를 main에 합쳐주세요" 라는 리뷰 요청을 만든다.
4
코드 리뷰 & 수정
팀원이 코멘트를 남기고, 요청에 맞춰 추가 커밋을 올린다.
5
Merge — 최종 합류
리뷰 통과 시 main 브랜치에 병합. 내 기여가 프로젝트에 공식 합류.

실생활에서
이렇게 쓰여요

우리가 매일 사용하는 앱과 서비스 속에 이 개념이 녹아 있다.

🐙
오픈소스 기여
React, Vue, Linux 등 오픈소스 프로젝트에 전 세계 개발자가 PR로 코드를 기여한다
🌐
GitHub Pages 포트폴리오
내 코드를 push하면 자동으로 웹사이트가 만들어진다. 무료 호스팅으로 포트폴리오 공개
⚡
자동 배포(Actions)
코드 merge 시 자동으로 테스트·빌드·배포까지 완료. 카카오, 네이버도 이 방식을 사용한다

💡 이 개념이 없다면 팀 개발은 이메일로 파일을 주고받는 혼란이 된다

자주 하는
오해 바로잡기

이렇게 생각하기 쉽지만, 실제로는 조금 다릅니다.

❌ 흔한 오해
  • GitHub=Git이다
  • GitHub에 올리면 누구나 볼 수 있다
  • GitHub의 Stars는 단순히 좋아요 버튼이다
✓ 실제로는
  • Git은 도구, GitHub는 Git 저장소를 호스팅하는 서비스다
  • private 저장소로 설정하면 초대된 사람만 볼 수 있다
  • Stars는 "관심 목록에 추가"에 가깝다. 개발자들이 나중에 참고하려고 저장하는 것

이걸 알면
AI와 이렇게 대화해요

이 개념을 이해하면 Claude나 ChatGPT를 훨씬 잘 활용할 수 있다.

👤
GitHub Pages로 내 포트폴리오 올리고 싶어요.
🤖
GitHub Pages는 무료로 정적 웹사이트를 호스팅할 수 있어요. 어떤 형식으로 만드셨나요?
👤
HTML/CSS로 만든 포트폴리오 파일이 있어요. GitHub Pages로 배포하는 단계별 방법을 처음부터 알려줘.
🤖
1) GitHub에서 [사용자명].github.io 이름으로 새 저장소 생성 → 2) HTML 파일을 push → 3) Settings > Pages에서 main 브랜치 선택 → 4) 몇 분 뒤 https://[사용자명].github.io 에서 확인하세요!
