언어 지형도 —
더 깊게 이해하기

프로그래밍 언어는 왜 이렇게 많을까? 각 언어는 특정 문제를 더 잘 풀기 위해 탄생했다.

1
목적이 달라서 만들어진다
웹을 위해 JavaScript, 데이터 과학을 위해 Python, 시스템 성능을 위해 C/C++이 탄생했다.
2
생산성과 성능은 트레이드오프
Python은 쓰기 쉽지만 느리다. C는 빠르지만 복잡하다. 둘 다 필요하기에 둘 다 살아있다.
3
생태계(라이브러리)가 언어를 키운다
Python이 AI의 표준이 된 건 TensorFlow·PyTorch 같은 강력한 라이브러리 덕분이다.
4
언어는 진화한다
JavaScript → TypeScript처럼 기존 언어의 단점을 보완하며 새 언어가 등장하기도 한다.

컴파일 vs 인터프리터

언어가 코드를 실제 실행하는 방식의 차이.

⚙️ 컴파일 언어
실행 전, 소스 전체를 기계어로 번역
실행 속도가 빠르다
예: C, C++, Go, Rust
배포 전 "빌드" 과정이 필요
📜 인터프리터 언어
줄 단위로 즉시 읽어 실행
즉시 피드백, 유연하게 실험 가능
예: Python, JavaScript, Ruby
런타임에 인터프리터가 필요

자주 하는
오해 바로잡기

이렇게 생각하기 쉽지만, 실제로는 조금 다릅니다.

❌ 흔한 오해
  • 어려운 언어를 쓸수록 실력 있는 개발자다
  • Python은 쉽고 JavaScript는 어렵다
  • 언어 하나 마스터하면 모든 걸 할 수 있다
✓ 실제로는
  • 목적에 맞는 언어를 선택하는 것이 진짜 실력이다
  • 둘 다 처음에는 어렵다. 목적에 따라 선택할 뿐
  • 언어마다 강점이 달라 여러 언어를 함께 쓰는 것이 일반적
