⚙️

프로세스 — 실행 중인 프로그램

SSD에 저장된 프로그램 파일이 RAM에 올라와 실제로 동작하는 상태.

📄 프로그램 (Program)
SSD에 저장된 파일
실행되지 않은 정적 상태
예: chrome.exe 파일
설계도 같은 존재
⚡ 프로세스 (Process)
RAM에 올라와 실행 중
자기만의 메모리 공간을 가짐
예: 지금 열린 Chrome 창
실제 살아 움직이는 존재
05 / 14

멀티태스킹 — 동시 실행의 비밀

여러 앱을 동시에 켜놓을 수 있는 이유 — CPU가 초고속으로 전환하기 때문이다.

🌐 Chrome
0.01초
→
🎵 음악 앱
0.01초
→
📝 메모장
0.01초
→
💬 카톡
0.01초
→
🌐 다시 Chrome
0.01초

실제로는 동시가 아니다. CPU 한 코어는 한 번에 한 프로세스만 처리하지만, 너무 빨라서 사람 눈에는 동시처럼 보인다

멀티코어 CPU는 진짜 동시 처리도 가능하다 — 8코어 = 8개 프로세스 병렬

06 / 14
📝

RAM —
프로세스가 살아 있는 곳

모든 프로세스는 RAM에 자기 영역을 할당받아 그 안에서 작업한다.

1
앱 실행 — SSD에서 RAM으로 복사
SSD의 프로그램 파일이 RAM의 빈 공간으로 로드된다.
2
CPU가 RAM에서 명령을 읽음
RAM은 빠른 접근이 가능해 CPU의 작업 속도를 따라잡을 수 있다.
3
작업 데이터도 RAM에 임시 저장
편집 중인 문서, 열린 탭, 입력 중인 텍스트 — 모두 RAM에.
4
앱 종료 시 RAM 영역 해제
프로세스가 끝나면 그 공간이 비워져 다른 프로세스가 사용 가능.
07 / 14

가상 메모리 — RAM이 부족할 때

RAM이 다 차면 OS가 SSD의 일부를 RAM처럼 빌려 쓴다. 이게 가상 메모리 (스왑).

📝 RAM
꽉 참
→
🔍 안 쓰는
프로세스 찾기
→
💾 SSD로
임시 이동
→
🆕 RAM 공간
확보

트레이드오프: 멈추진 않지만 SSD는 RAM보다 100배 느리다 — 컴퓨터가 매우 느려진다

RAM이 충분하면 가상 메모리를 안 쓰니까, RAM 용량이 큰 게 결국 빠르다

메모리 누수 —
조용히 새는 RAM

프로그램이 RAM을 사용한 뒤 반환하지 않아, 계속 쌓이는 현상.

📈
증상
시간이 지날수록 컴퓨터가 점점 느려진다.
특정 앱의 RAM 사용량이 계속 늘어남
🐞
원인
개발자가 메모리 해제를 빠뜨린 버그.
오래된 브라우저 탭이 자주 원인
🔄
해결
해당 앱 재시작 또는
최악의 경우 컴퓨터 재부팅
09 / 14
📊

작업 관리자 —
프로세스 들여다보기

지금 내 컴퓨터에서 어떤 프로세스가 RAM을 쓰는지 확인해보자.

1
Windows: Ctrl + Shift + Esc
macOS: 활성 상태 보기 (Activity Monitor) 실행.
2
"프로세스" 탭 보기
실행 중인 모든 프로세스와 각각의 CPU·메모리 사용량이 표시된다.
3
메모리 사용량 정렬
메모리 컬럼 클릭 → RAM을 가장 많이 쓰는 프로세스가 위로 올라온다.
4
불필요한 프로세스 종료
의심되는 항목 우클릭 → "작업 끝내기"로 RAM 회수.

이걸 알면
AI와 이렇게 대화해요

프로세스 개념을 알면 컴퓨터 트러블슈팅이 훨씬 정확해진다.

👤
컴퓨터가 너무 느려요
🤖
어떤 상황에서 느린가요? 부팅, 특정 앱, 또는 전반적으로?
👤
Chrome 탭을 30개 정도 띄우면 메모리가 90% 이상이 되고 디스크가 100%로 튀어요. 가상 메모리 때문인 것 같은데 어떻게 해결하나요?
🤖
정확한 진단입니다! RAM이 부족해 가상 메모리(스왑)가 작동 중이에요. 해결책: (1) 안 쓰는 탭 닫기 (2) Chrome 메모리 절약 모드 켜기 (3) 장기적으로 RAM 증설.

자주 하는
오해 바로잡기

이렇게 생각하기 쉽지만, 실제로는 조금 다릅니다.

❌ 흔한 오해
  • 프로그램과 프로세스는 같은 말이다
  • RAM이 90% 차면 무조건 추가해야 한다
  • 여러 앱을 켜면 진짜 동시에 실행된다
✓ 실제로는
  • 프로그램은 파일, 프로세스는 실행된 인스턴스다
  • OS는 RAM을 적극 활용하도록 설계됐다. 캐시도 사용량에 포함
  • 1코어는 빠른 전환, 멀티코어만 진짜 병렬 처리한다
