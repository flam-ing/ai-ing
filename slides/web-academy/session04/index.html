인터넷 vs WWW

흔히 같은 말로 쓰지만, 완전히 다른 개념이다.

🌐 인터넷 (Internet)
전 세계 컴퓨터를 잇는 인프라
물리적인 케이블·라우터·기지국
위에서 다양한 서비스가 동작
이메일, 게임, 화상통화 모두 인터넷 위
🕸️ WWW (Web)
인터넷 위에서 동작하는 한 가지 서비스
웹 브라우저로 보는 페이지들의 집합
HTTP 프로토콜과 HTML 문서 기반
1989년 팀 버너스리가 발명

DNS —
인터넷의 전화번호부

사람은 이름을 외우고, 컴퓨터는 숫자(IP)로 통신한다. 그 사이를 변환해주는 게 DNS.

1
사용자가 google.com 입력
브라우저는 도메인 이름만 받았을 뿐, 실제 서버 위치는 모른다.
2
DNS 서버에 질의
"google.com의 IP가 뭐야?" → DNS 서버가 응답: 142.250.x.x
3
실제 IP로 접속
브라우저는 받은 IP로 서버에 연결해 페이지를 받아온다.
4
결과 표시
사용자는 도메인만 보고도 빠르게 사이트를 이용할 수 있다.

🔗

URL —
한 줄에 담긴 정보

주소 표시줄의 한 줄짜리 텍스트, 사실은 네 가지 정보가 담겨있다.

https://www.example.com/products/shoes?color=red&size=10
프로토콜 — 통신 방식 (https = 암호화)
도메인 — 서버의 이름 주소
경로 — 서버 안의 자원 위치
쿼리 — 추가 조건/필터 (? 이후)

자주 하는
오해 바로잡기

이렇게 생각하기 쉽지만, 실제로는 조금 다릅니다.

❌ 흔한 오해
  • 인터넷 = 웹(WWW)이다
  • IP 주소는 평생 같은 번호다
  • Wi-Fi가 곧 인터넷이다
✓ 실제로는
  • 웹은 인터넷 위 하나의 서비스. 이메일·게임도 인터넷 서비스다
  • 가정용 IP는 동적이라 자주 바뀐다. 서버용은 고정 IP를 쓴다
  • Wi-Fi는 무선 연결 방식. 공유기가 인터넷에 연결돼야 인터넷이 된다
