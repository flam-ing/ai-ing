써야 할까?

npm install —
더 깊게 이해하기

명령어 하나를 입력하면 뒤에서는 이런 일이 순서대로 일어난다.

1
package.json 읽기
어떤 패키지가 필요한지, 각각 어떤 버전이 요구되는지 목록을 확인한다.
2
의존성 트리 분석
A 패키지가 B를 필요로 하고, B가 C를 필요로 하는 중첩 의존성을 모두 계산한다.
3
npm 레지스트리에서 다운로드
npmjs.com에서 필요한 패키지들을 버전에 맞게 다운로드한다.
4
node_modules 폴더에 저장
모든 패키지가 node_modules 폴더에 저장된다. 수백 개가 쌓여 수백 MB가 되기도 한다.

버전 충돌 —
실제로는 어떻게?

패키지 A가 lodash 4를 요구하고, 패키지 B가 lodash 3을 요구하면 충돌이 일어난다.

📄 A requires
lodash@4
→
📄 B requires
lodash@3
→
💥 버전 충돌
발생
→
🔧 lock 파일로
해결

핵심: package-lock.json(npm) 또는 pnpm-lock.yaml이 정확한 버전을 고정해 어디서든 같은 환경을 보장한다

08 / 15

핵심 파일 — package.json

프로젝트의 신분증 같은 파일이다.

1
어떤 패키지가 필요한지 목록
dependencies 항목에 내 프로젝트가 의존하는 패키지들이 버전과 함께 적힌다.
2
자주 쓰는 명령어 저장
scripts에 dev, build 같은 짧은 이름을 등록해 실행.
3
node_modules 폴더
설치된 실제 패키지 파일들이 저장되는 곳. 수 GB가 되기도 해서 Git에 올리지 않는다.
4
재생성 가능
다른 PC에서도 pnpm install 한 번이면 동일한 환경 복원.

10 / 15
💡

자주 하는
오해 바로잡기

이렇게 생각하기 쉽지만, 실제로는 조금 다릅니다.

❌ 흔한 오해
  • 패키지=앱이다
  • npm=Node.js이다
  • package.json을 직접 편집하면 안 된다
✓ 실제로는
  • 패키지는 코드 라이브러리다. 실행되는 앱이 아니라 다른 코드에서 불러다 쓰는 모듈이다
  • npm은 Node.js에 포함된 패키지 관리자다. Node.js는 런타임, npm은 도구다
  • package.json은 텍스트 파일로 직접 편집 가능하다. 단, 형식(JSON)을 지켜야 한다
