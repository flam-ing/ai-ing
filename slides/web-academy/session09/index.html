왜 운영체제가
필요한가?

컴퓨터 부품들은 스스로 일하지 못한다. 누군가가 중간에서 지휘해야 한다.
프로그램들이 CPU와 메모리를 서로 다투지 않게 공정하게 분배해야 한다.

바로 이 역할을 하는 것이 운영체제(OS)다.
우리가 앱을 더블클릭만 하면 실행되는 이유 — OS가 뒤에서 모든 걸 처리하기 때문이다.

04 / 15

OS의 3가지 핵심 역할

운영체제는 컴퓨터의 총괄 매니저다. 크게 세 가지 일을 한다.

1
🔩 하드웨어 자원 관리
CPU 시간, 메모리, 디스크, 네트워크 등을 여러 프로그램에게 공평하게 나눠준다.
2
💿 프로그램 실행 환경 제공
앱이 돌아갈 수 있는 표준 인터페이스(시스템 콜)를 제공한다. 개발자는 HW를 몰라도 앱을 만들 수 있다.
3
🖱️ 사용자 인터페이스 제공
GUI(창/아이콘) 또는 CLI(터미널) 같은 사용자와의 대화 창구를 제공한다.
05 / 15
❌

OS의 3가지 역할 —
더 깊게 이해하기

OS가 없다면 우리가 당연하게 쓰는 것들이 전부 불가능해진다.

1
앱 실행 불가
프로그램은 CPU·메모리를 직접 예약할 수 없다. 실행 환경 자체가 사라진다.
2
파일 저장 불가
파일 시스템(HFS+, NTFS 등)은 OS가 관리한다. OS 없이는 데이터를 디스크에 쓸 방법이 없다.
3
인터넷 불가
네트워크 드라이버도 OS 위에서 동작한다. 브라우저는 물론 ping조차 보낼 수 없다.
4
보안 완전 붕괴
어떤 프로그램도 다른 프로그램의 메모리를 마음대로 읽고 쓸 수 있게 된다.

08 / 15

OS의 심장 — 커널(Kernel)

커널은 OS의 가장 깊은 곳에서 항상 동작하는 핵심 프로그램이다.

👤 사용자 앱
Chrome, VS Code
↓
🧩 커널
시스템 콜 처리
↓
🔩 하드웨어
CPU, RAM, SSD

앱이 "파일을 읽어줘"라고 요청하면, 커널이 디스크에 직접 명령을 내려 데이터를 가져온다.
커널은 항상 메모리에 상주하며, 전원이 켜진 동안 한순간도 쉬지 않는다.

09 / 15
📱

실생활에서
이렇게 쓰여요

우리가 매일 사용하는 앱과 서비스 속에 이 개념이 녹아 있다.

🔔
스마트폰 알림 관리
Android OS가 카카오톡·유튜브 알림 우선순위를 조율하고 배터리 사용량을 배분한다.
🔍
macOS Spotlight 검색
파일명, 앱, 계산, 사전까지 한 번에 — OS가 파일 시스템 전체를 인덱싱해 실시간 검색한다.
📊
Windows 작업 관리자
OS가 실시간으로 각 프로그램의 CPU·메모리 점유율을 추적하고 강제 종료도 가능하게 한다.

💡 이 개념이 없다면 한 앱이 CPU를 독점해 다른 앱이 완전히 멈춰버린다.

💡

자주 하는
오해 바로잡기

이렇게 생각하기 쉽지만, 실제로는 조금 다릅니다.

❌ 흔한 오해
  • OS = Windows 뿐이다
  • OS가 없어도 앱은 실행된다
  • OS 업데이트는 선택사항이다
✓ 실제로는
  • macOS, Linux, Android, iOS도 모두 OS다
  • OS 없이는 어떤 앱도 실행될 수 없다
  • 보안 패치가 포함돼 업데이트는 필수다

✅

이렇게
이해하셨나요?

✅
맞게 이해하셨어요
OS는 하드웨어와 앱 사이의 관리자다 — OS가 없으면 앱은 HW에 직접 접근할 방법이 없다.
❌
이렇게는 아니에요
앱이 직접 하드웨어를 제어한다 — 앱은 OS에 요청(시스템 콜)만 할 뿐, 직접 제어하지 않는다.
✅
맞게 이해하셨어요
커널은 OS의 핵심으로 항상 메모리에 올라와 있다 — 전원이 켜진 순간부터 꺼질 때까지 상주한다.
❌
이렇게는 아니에요
OS는 Windows뿐이다 — macOS, Linux, Android, iOS 모두 엄연한 운영체제다.

💪 OS의 역할을 이해한 것만으로도 개발자 사고방식의 절반은 완성됐습니다!
