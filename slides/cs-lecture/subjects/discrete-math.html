이산수학

이산수학은 컴퓨터의 언어로 된 수학입니다. 0과 1, 참과 거짓, 연결과 단절처럼 딱딱 떨어지는 것을 다루기에, 모든 전공 과목의 밑바탕이 됩니다.

이산수학이란?

이산수학(discrete mathematics)의 "이산(離散)"은 서로 떨어져 있어 셀 수 있는 것이라는 뜻입니다. 사람 수, 동전의 앞뒤, 켜짐/꺼짐처럼 중간값 없이 똑똑 끊어지는 대상을 다룹니다. 반대로 길이·온도처럼 매끄럽게 이어지는 값을 다루는 것은 연속수학(미적분)입니다.

컴퓨터는 0과 1, 즉 이산적인 값으로 작동합니다. 그래서 이산수학은 자료구조·알고리즘·데이터베이스·암호학 등 컴퓨터공학 거의 모든 분야의 공통 도구가 됩니다.

📌 이 페이지에서 배우는 것

명제와 논리 · 집합 · 관계와 함수 · 조합론(경우의 수) · 그래프 이론 · 수학적 귀납법을 차례로 다룹니다. 진리표·벤다이어그램·그래프 그림으로 직관을 키웁니다.

명제와 논리

명제(proposition)는 참(True) 또는 거짓(False)을 명확히 가릴 수 있는 문장입니다. "3은 홀수다"는 참인 명제, "5 > 10"은 거짓인 명제입니다. "오늘 날씨 좋다"처럼 사람마다 다르게 판단하는 문장은 명제가 아닙니다.

명제들을 논리 연산자로 묶어 더 복잡한 판단을 만듭니다. 컴퓨터의 조건문이 바로 이 논리에서 나옵니다.

  • AND(그리고, ∧) — 둘 다 참이어야 참
  • OR(또는, ∨) — 하나라도 참이면 참
  • NOT(부정, ¬) — 참은 거짓으로, 거짓은 참으로 뒤집음

각 경우의 결과를 모두 적은 표를 진리표(truth table)라고 합니다.

AND 진리표: 둘 다 참(T)일 때만 결과가 참 — 초록 칸 한 줄뿐
logic.py
P = True
Q = False

print(P and Q)   # False  — 둘 다 참이어야 참
print(P or Q)    # True   — 하나라도 참이면 참
print(not P)     # False  — 참을 뒤집으면 거짓

# 로그인 조건: 아이디가 있고(and) 비밀번호가 맞아야
can_login = (id_exists) and (password_ok)

집합

집합(set)은 서로 다른 원소들의 모음입니다. 중복은 허용하지 않고 순서도 따지지 않습니다. {1, 2, 3}과 {3, 2, 1}은 같은 집합입니다. 집합끼리는 합치고(합집합), 겹치는 부분을 찾고(교집합), 빼는(차집합) 연산을 합니다.

두 원이 겹치는 가운데가 교집합(A ∩ B), 둘을 합친 전체가 합집합(A ∪ B)
연산기호의미
합집합A ∪ BA나 B에 속한 모든 원소
교집합A ∩ BA와 B 양쪽에 다 있는 원소
차집합A − BA에는 있고 B에는 없는 원소
부분집합A ⊆ BA의 모든 원소가 B에도 있음
sets.py
A = {1, 2, 3}
B = {3, 4, 5}

print(A | B)   # {1, 2, 3, 4, 5}  합집합
print(A & B)   # {3}              교집합
print(A - B)   # {1, 2}           차집합
print(3 in A)  # True             원소인지 확인
💡 어디에 쓰일까

데이터베이스에서 "두 조건을 모두 만족하는 회원"을 찾는 일이 바로 교집합입니다. SQL의 JOIN·UNION도 집합 연산의 응용입니다.

관계와 함수

관계(relation)는 두 집합의 원소를 짝지어 연결한 것입니다. "학생 ↔ 수강과목"처럼 무엇이 무엇과 이어지는지를 나타냅니다. 한 학생이 여러 과목을 들을 수 있으니, 관계에서는 하나가 여러 개와 연결될 수 있습니다.

함수(function)는 관계 중에서도 특별한 규칙을 지킵니다. 입력 하나에 출력이 정확히 하나씩 대응해야 합니다. "사람 → 주민번호"는 함수지만, "사람 → 친구"는 친구가 여럿일 수 있어 함수가 아닙니다.

함수: 왼쪽 각 원소에서 화살표가 정확히 하나씩 나감 (a→1, b→1, c→3 모두 OK)
⚠️ 함수의 조건

한 입력에서 화살표가 두 개 이상 나가면 함수가 아닙니다. 단, 서로 다른 입력이 같은 출력으로 가는 것(a→1, b→1)은 함수로 허용됩니다.

조합론 (경우의 수)

조합론(combinatorics)은 "몇 가지 경우가 가능한가"를 세는 분야입니다. 비밀번호 가짓수, 카드 패의 수, 알고리즘이 검사해야 할 경우의 수 등을 계산할 때 씁니다. 핵심은 순열과 조합의 구분입니다.

  • 순열(Permutation) — 순서가 중요한 나열. (1, 2)와 (2, 1)을 다르게 셈. 예: 달리기 1·2등.
  • 조합(Combination) — 순서는 상관없이 뽑기만. {1, 2}와 {2, 1}을 같게 셈. 예: 대표 2명 선발.
구분공식예: 3개 중 2개
순열 (순서 O)nPr = n! / (n−r)!3P2 = 6
조합 (순서 X)nCr = n! / (r!(n−r)!)3C2 = 3
counting.py
from math import perm, comb

print(perm(3, 2))   # 6  — (1,2)(2,1)(1,3)(3,1)(2,3)(3,2)
print(comb(3, 2))   # 3  — {1,2} {1,3} {2,3}

# 4자리 숫자 비밀번호(0~9 중복 허용)의 가짓수
print(10 ** 4)      # 10000 — 곱의 법칙: 각 자리 10가지
💡 헷갈릴 땐

"순서를 바꾸면 다른 경우인가?"를 물어보세요. 다르면 순열, 같으면 조합입니다.

그래프 이론 기초

그래프(graph)는 점(정점, vertex)들을 선(간선, edge)으로 이은 그림입니다. "무엇과 무엇이 연결되어 있는가"를 표현하는 데 최고의 도구입니다. 지하철 노선도, SNS 친구 관계, 인터넷 링크가 모두 그래프입니다.

정점(원) 4개와 간선(선) 4개로 이뤄진 그래프 — A는 B, C와 연결

그래프를 컴퓨터에 저장할 때는 흔히 인접 리스트(각 정점이 연결된 이웃 목록을 들고 있음)를 씁니다.

graph.py
# 인접 리스트로 그래프 표현
graph = {
    "A": ["B", "C"],
    "B": ["A", "C", "D"],
    "C": ["A", "B"],
    "D": ["B"],
}

print(graph["B"])        # ['A', 'C', 'D']  — B의 이웃
print(len(graph["B"]))   # 3  — B의 차수(연결된 간선 수)
📌 어디서 쓰나

최단 경로 찾기(내비게이션), 친구 추천, 작업 순서 정하기(위상 정렬) 등 수많은 알고리즘이 그래프 위에서 돕니다. 2학년 자료구조·알고리즘에서 본격적으로 다룹니다.

수학적 귀납법

수학적 귀납법(mathematical induction)은 "모든 자연수에 대해 어떤 명제가 참이다"를 증명하는 강력한 방법입니다. 도미노 쓰러뜨리기에 비유하면 완벽합니다.

  1. 기초 단계 — 첫 번째 도미노(n=1)가 쓰러짐을 확인한다.
  2. 귀납 단계 — "k번째가 쓰러지면 k+1번째도 쓰러진다"를 보인다.

이 둘이 모두 참이면, 첫 도미노가 쓰러지는 순간 그 다음, 또 그 다음… 끝없이 모두 쓰러집니다. 즉 모든 자연수에 대해 명제가 참이 됩니다.

기초(첫 도미노) + 귀납(연쇄 규칙) = 모두 쓰러짐
induction.py
# 명제: 1 + 2 + ... + n = n(n+1)/2
def sum_formula(n):
    return n * (n + 1) // 2

# 귀납법으로 증명한 공식을 직접 검증
for n in range(1, 6):
    직접합 = sum(range(1, n + 1))
    print(n, 직접합, sum_formula(n))   # 두 값이 항상 같음
💡 컴퓨터공학과의 연결

귀납법의 사고방식은 재귀(recursion)와 똑같습니다. "작은 문제를 풀 수 있으면 큰 문제도 풀린다" — 알고리즘 정당성 증명에 자주 쓰입니다.

논리 동치와 드모르간 법칙

겉보기엔 다른 두 논리식이 모든 경우에 똑같은 참/거짓을 내면, 두 식은 논리적으로 동치(equivalent)입니다(기호 ≡). 동치는 복잡한 조건을 더 단순하게 바꿔 코드를 깔끔하게 만드는 데 쓰입니다.

가장 유명한 동치가 드모르간 법칙(De Morgan's laws)입니다. "부정을 안으로 넣으면 AND와 OR가 뒤집힌다"고 외우면 됩니다.

📌 드모르간 법칙

¬(P ∧ Q) ≡ (¬P) ∨ (¬Q)
¬(P ∨ Q) ≡ (¬P) ∧ (¬Q)

"비가 오고(AND) 추운 게 아니다"는 "비가 안 오거나(OR) 안 춥다"와 같은 말입니다. 진리표로 두 식의 마지막 열이 완전히 같음을 확인할 수 있습니다.

두 식의 결과 열(강조 표시)이 네 줄 모두 일치 → 논리적으로 동치
de_morgan.py
# 드모르간 법칙을 모든 경우에 대해 검증
for P in (True, False):
    for Q in (True, False):
        left  = not (P and Q)
        right = (not P) or (not Q)
        print(P, Q, left == right)   # 항상 True → 동치임

# 실전: 조건을 더 읽기 쉽게 바꾸기
# not (logged_in and is_admin)  ==  (not logged_in) or (not is_admin)

술어논리와 양화사 (∀·∃)

"x는 짝수다"처럼 변수 x에 따라 참/거짓이 갈리는 문장을 술어(predicate) P(x)라 합니다. 술어 하나만으로는 참/거짓이 정해지지 않습니다 — x가 정해져야 합니다. 여기에 "얼마나 많은 x에 대해 참인가"를 붙이는 기호가 양화사(quantifier)입니다.

기호이름읽는 법참이 되려면
∀x P(x)전칭(全稱)"모든 x에 대해"예외 하나도 없어야 참
∃x P(x)존재(存在)"어떤 x가 존재해"하나라도 있으면 참

둘은 부정으로 서로 연결됩니다(양화사판 드모르간): ¬(∀x P(x)) ≡ ∃x ¬P(x) — "모두가 P인 게 아니다"는 "P가 아닌 게 하나 있다"와 같습니다.

∀는 단 하나의 예외도 없어야 참, ∃는 단 하나만 있으면 참
quantifier.py
nums = [2, 4, 6, 8]

print(all(x % 2 == 0 for x in nums))   # ∀: 모두 짝수? → True
print(any(x > 7 for x in nums))        # ∃: 7보다 큰 게 있나? → True
print(all(x > 7 for x in nums))        # False — 예외(2,4,6)가 있음
💡 파이썬의 all / any

파이썬의 all()은 전칭(∀), any()는 존재(∃)와 정확히 같습니다. 양화사가 추상적으로 느껴진다면 이 두 함수를 떠올리세요.

수론 기초: 소수·최대공약수·모듈러

수론(number theory)은 정수의 성질을 다루는 분야로, 오늘날 암호학의 바탕입니다. 세 가지 핵심 개념을 봅시다.

  • 소수(prime) — 1과 자기 자신으로만 나누어지는 수(2, 3, 5, 7, 11…). 큰 소수는 암호 키의 재료입니다.
  • 최대공약수(GCD) — 두 수를 동시에 나누는 가장 큰 수. 유클리드 호제법으로 빠르게 구합니다.
  • 모듈러 연산(mod) — 나눗셈의 나머지. 시계(12시간)처럼 일정 범위에서 돌고 도는 계산입니다.
mod 6: 0,1,2,3,4,5 다음은 다시 0 — 12 mod 6 = 0, 7 mod 6 = 1
number_theory.py
from math import gcd

# 모듈러 연산 (나머지)
print(17 % 5)     # 2   — 17을 5로 나눈 나머지
print(14 % 7)     # 0   — 나누어떨어짐 → 7은 14의 약수

# 최대공약수
print(gcd(12, 18))   # 6

# 소수 판별 (2부터 √n까지 나눠 보기)
def is_prime(n):
    if n < 2:
        return False
    for d in range(2, int(n ** 0.5) + 1):
        if n % d == 0:
            return False
    return True

print([n for n in range(2, 20) if is_prime(n)])   # [2,3,5,7,11,13,17,19]
💡 어디에 쓰일까

인터넷 결제·메신저 암호화에 쓰이는 RSA는 "두 큰 소수를 곱하긴 쉬워도, 그 곱을 다시 소수로 쪼개긴 매우 어렵다"는 수론의 성질에 기댑니다. 해시 테이블의 인덱스 계산에도 mod가 등장합니다.

점화식과 재귀관계

점화식(recurrence relation)은 어떤 항을 앞의 항(들)로 정의하는 규칙입니다. "다음 값은 이전 값으로 이렇게 계산한다"는 식이라, 자연스럽게 컴퓨터의 재귀(recursion)로 옮겨집니다.

가장 유명한 예가 피보나치 수열입니다. 시작 두 항을 정하고(기저), 그 뒤는 앞 두 항의 합으로 정의합니다.

📌 피보나치 점화식

F(0) = 0, F(1) = 1 (기저 조건)
F(n) = F(n−1) + F(n−2) (n ≥ 2)

0, 1로 시작 — 그 뒤는 바로 앞 두 수를 더한 값 (1+2=3, 2+3=5, …)
recurrence.py
# 점화식을 그대로 재귀 함수로 옮김
def fib(n):
    if n < 2:            # 기저 조건: F(0)=0, F(1)=1
        return n
    return fib(n - 1) + fib(n - 2)   # 점화식

print([fib(n) for n in range(8)])   # [0, 1, 1, 2, 3, 5, 8, 13]
⚠️ 기저 조건은 필수

재귀에는 반드시 멈추는 지점(기저 조건)이 있어야 합니다. 없으면 함수가 영원히 자기 자신을 부르다 멈춥니다. 점화식의 시작 항이 바로 그 기저 조건입니다.

확률 기초

확률(probability)은 어떤 일이 일어날 가능성을 0과 1 사이의 수로 나타낸 것입니다. 0이면 절대 안 일어남, 1이면 반드시 일어남입니다. 모든 경우가 똑같이 가능할 때, 확률은 간단한 분수로 구합니다.

📌 기본 공식

P(사건) = 원하는 경우의 수 ÷ 전체 경우의 수

주사위 한 개를 던질 때 "짝수가 나올" 확률은 짝수(2,4,6) 3가지 ÷ 전체 6가지 = 1/2 입니다. 앞서 배운 조합론(경우의 수)이 분자·분모를 세는 데 그대로 쓰입니다.

개념의미예 (주사위)
여사건일어나지 않을 확률 = 1 − P짝수 아닐 확률 = 1 − 1/2 = 1/2
독립 사건둘 다 일어날 확률 = P × P두 번 다 6 = 1/6 × 1/6
확률은 0(불가능)에서 1(확실) 사이의 값 — 주사위 짝수는 정확히 가운데
probability.py
import random

# 이론 확률: 짝수 3가지 / 전체 6가지
print(3 / 6)        # 0.5

# 실험으로 확인: 주사위를 10000번 던져 보기
rolls = [random.randint(1, 6) for _ in range(10000)]
evens = [r for r in rolls if r % 2 == 0]
print(len(evens) / len(rolls))   # 0.5 근처 — 많이 던질수록 이론값에 가까워짐
💡 큰 수의 법칙

적게 던지면 들쭉날쭉하지만, 아주 많이 반복하면 실제 비율이 이론 확률에 가까워집니다. 머신러닝·시뮬레이션이 데이터를 많이 쓰는 이유와 닿아 있습니다.

부울 대수와 논리회로

앞서 배운 명제 논리(AND·OR·NOT)를 0과 1로 계산하는 대수로 본 것이 부울 대수(Boolean algebra)입니다. 참=1, 거짓=0으로 두면 논리가 곧 계산이 됩니다. 이 계산을 전기 신호로 구현한 부품이 논리 게이트(logic gate)이고, 게이트를 이어 붙인 것이 논리회로 — 즉 CPU의 가장 밑바닥 재료입니다.

논리 게이트 기호: 두 입력을 받아 AND·OR로 계산하고, NOT은 신호를 뒤집음
ABAND (A·B)OR (A+B)

0101
1001
1111
boolean.py
# 0/1 비트로 보는 부울 대수 (& = AND, | = OR, ^ = XOR)
print(1 & 0)   # 0  — AND
print(1 | 0)   # 1  — OR
print(1 ^ 1)   # 0  — XOR(다르면 1, 같으면 0)

# True/False 도 1/0 처럼 계산됨
print(True + True)   # 2
📌 논리 → 회로 → 컴퓨터

덧셈조차 AND·OR·XOR 게이트의 조합(가산기)으로 만들어집니다. 이산수학의 명제 논리가 그대로 3학년 컴퓨터구조의 회로 설계로 이어집니다.

비둘기집 원리

비둘기집 원리(pigeonhole principle)는 너무 당연해 보이지만 강력한 도구입니다. n+1마리의 비둘기를 n개의 집에 넣으면, 적어도 한 집에는 2마리 이상이 들어간다는 원리입니다. "물건이 칸보다 많으면 어딘가는 겹친다"는 말입니다.

집(3)보다 비둘기(4)가 많으면 어느 한 집엔 2마리 이상 — 피할 수 없음

응용 예: "한국인 중 머리카락 수가 똑같은 두 사람이 반드시 있다"(머리카락 수의 범위보다 인구가 많으므로), "13명만 모이면 생일 달이 같은 두 사람이 있다"(달은 12개뿐).

pigeonhole.py
# 13명의 생일 달(1~12)을 모으면 반드시 겹치는 달이 있다
import random
months = [random.randint(1, 12) for _ in range(13)]
print(len(months), len(set(months)))   # 13 vs 12 이하 → 중복 존재 보장

# set 으로 줄였을 때 인원보다 작으면 겹친 것
print("겹치는 달 있음:", len(set(months)) < len(months))   # 항상 True
💡 컴퓨터공학에서

해시 테이블에서 키가 칸보다 많으면 충돌(collision)이 반드시 생긴다는 사실이 바로 비둘기집 원리입니다. 무손실 압축이 모든 파일을 줄일 수 없다는 증명에도 쓰입니다.

🎯 핵심 정리
  • 이산수학은 똑똑 끊어지는(이산적인) 대상을 다루며 컴퓨터공학의 공통 도구다.
  • 명제는 참/거짓을 가리는 문장이고, AND·OR·NOT을 진리표로 정리한다.
  • 집합은 중복 없는 모음이며 합집합·교집합·차집합으로 다룬다.
  • 함수는 입력 하나에 출력 하나가 대응하는 특별한 관계다.
  • 조합론은 순서를 따지면 순열, 안 따지면 조합으로 경우의 수를 센다.
  • 그래프는 정점과 간선으로 연결 관계를, 귀납법은 도미노처럼 모든 자연수를 증명한다.
  • 드모르간 법칙은 부정을 안으로 넣으며 AND·OR를 뒤집고, 양화사 ∀(모두)·∃(존재)는 술어의 적용 범위를 정한다.
  • 수론(소수·GCD·모듈러)은 암호의 바탕이고, 점화식은 앞 항으로 다음 항을 정의해 재귀로 옮겨진다.
  • 확률은 원하는 경우 ÷ 전체 경우, 부울 대수는 논리를 0·1 계산(게이트)으로 본다.
  • 비둘기집 원리: 칸보다 물건이 많으면 어딘가는 반드시 겹친다(해시 충돌의 근거).
명제(Proposition)
참 또는 거짓을 명확히 가릴 수 있는 문장.
진리표(Truth Table)
모든 입력 조합에 대한 논리식의 참/거짓을 적은 표.
집합(Set)
중복 없이, 순서 없이 모은 서로 다른 원소들의 모임.
함수(Function)
입력 하나에 출력이 정확히 하나씩 대응하는 관계.
그래프(Graph)
정점(점)들을 간선(선)으로 연결해 관계를 나타낸 구조.
수학적 귀납법
기초 단계와 귀납 단계로 모든 자연수에 대한 명제를 증명하는 방법.
논리 동치(≡)
모든 경우에 참/거짓이 같은 두 논리식의 관계.
드모르간 법칙
¬(P∧Q)≡¬P∨¬Q 등, 부정을 분배하며 AND·OR를 뒤집는 규칙.
양화사(∀·∃)
술어가 모든 대상(∀)에/어떤 대상(∃)에 참인지를 나타내는 기호.
모듈러 연산(mod)
나눗셈의 나머지로 일정 범위에서 순환하는 정수 연산.
점화식
한 항을 앞의 항(들)로 정의하는 식. 재귀로 직접 옮겨짐.
부울 대수
참/거짓을 1/0으로 두고 논리를 계산하는 대수. 논리회로의 바탕.
비둘기집 원리
칸보다 대상이 많으면 한 칸에 둘 이상이 들어간다는 원리.
