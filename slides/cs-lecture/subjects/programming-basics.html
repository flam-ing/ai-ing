프로그래밍 기초

프로그래밍은 컴퓨터에게 일을 시키는 글쓰기입니다. 컴퓨터가 알아듣는 문장 몇 가지만 익히면, 누구나 자기 생각을 코드로 옮길 수 있습니다.

프로그래밍이란?

프로그래밍(programming)은 컴퓨터가 따라 할 순서가 정해진 명령 목록을 만드는 일입니다. 이 명령 목록을 프로그램이라 하고, 사람과 컴퓨터가 둘 다 알아볼 수 있게 약속한 언어를 프로그래밍 언어라고 합니다.

요리 레시피에 비유하면 쉽습니다. "물을 끓인다 → 면을 넣는다 → 3분 기다린다"처럼, 컴퓨터도 위에서 아래로 한 줄씩 차례대로 명령을 실행합니다. 컴퓨터는 똑똑하지 않고 시킨 그대로만 하므로, 빠뜨리거나 순서가 틀리면 엉뚱한 결과가 나옵니다.

데이터를 받아(입력) → 가공하고(처리) → 결과를 보여줌(출력)
📌 이 페이지에서 배우는 것

변수와 자료형 · 연산자와 입출력 · 조건문 · 반복문 · 함수, 그리고 오류를 잡는 디버깅까지 다룹니다. 예제는 입문용으로 가장 인기 있는 파이썬(Python)으로 보여줍니다.

변수와 자료형

변수(variable)는 값을 담아 두는 "이름표 붙은 상자"입니다. 상자에 이름을 붙여 두면, 나중에 그 이름으로 안의 값을 꺼내 쓰거나 새 값으로 바꿀 수 있습니다.

상자(변수) 위에 이름표, 안에는 값. age 상자엔 숫자, name 상자엔 글자

자료형(data type)은 상자 안에 든 값의 종류입니다. 숫자인지 글자인지에 따라 컴퓨터가 다루는 방법이 달라집니다. 자주 쓰는 기본 자료형은 다음과 같습니다.

자료형이름예시 값
int정수0, 29, -7
float실수(소수)3.14, -0.5
str문자열(글자)"안녕", "Python"
bool참/거짓True, False
variables.py
age = 29              # int  (정수)
height = 175.5        # float(소수)
name = "민수"          # str  (문자열)
is_student = True     # bool (참/거짓)

age = age + 1         # 상자의 값을 바꿔 다시 담기 → 30
print(type(name))     # <class 'str'>  — 자료형 확인
💡 팁

파이썬은 age = 29처럼 값을 넣으면 자료형을 알아서 판단합니다. 변수 이름은 user_age처럼 내용을 알 수 있게 짓는 습관이 중요합니다.

연산자와 입출력

연산자(operator)는 값을 계산하거나 비교하는 기호입니다. 사칙연산은 물론, 두 값을 비교해 참/거짓을 만드는 비교 연산도 있습니다.

분류연산자의미
산술+ - * /더하기·빼기·곱하기·나누기
산술%, **나머지, 거듭제곱
비교== != > <같다·다르다·크다·작다 (결과는 참/거짓)
논리and or not그리고·또는·아니다

입출력은 프로그램이 사람과 대화하는 통로입니다. input()으로 사용자에게 값을 받고, print()로 결과를 화면에 보여줍니다.

io_calc.py
name = input("이름을 입력하세요: ")   # 사용자 입력 받기(항상 문자열)
year = int(input("태어난 연도: "))    # 숫자로 쓰려면 int()로 변환

age = 2026 - year
print(name + "님은 올해", age, "살입니다.")
print("내후년이면", age + 2, "살이 되겠네요!")
⚠️ 흔한 실수

input()으로 받은 값은 항상 문자열입니다. 숫자 계산을 하려면 int()나 float()로 바꿔야 합니다. 안 그러면 "3" + "4"가 34(글자 붙이기)가 됩니다.

조건문 (if)

조건문은 "만약 ~라면 이렇게, 아니면 저렇게" 하고 상황에 따라 다른 길로 가는 갈림길입니다. 비교 연산의 결과(참/거짓)에 따라 실행할 코드가 갈립니다.

조건이 참이면 왼쪽, 거짓이면 오른쪽 길로 — 둘 중 하나만 실행
grade.py
score = int(input("점수: "))

if score >= 90:
    print("A 학점")
elif score >= 60:          # 위 조건이 거짓일 때만 검사
    print("합격")
else:                       # 어느 조건에도 안 맞으면
    print("불합격")
⚠️ 들여쓰기가 곧 문법

파이썬은 들여쓰기(보통 띄어쓰기 4칸)로 "이 코드가 if에 속한다"를 표시합니다. 들여쓰기가 어긋나면 오류가 나거나 의도와 다르게 동작합니다.

반복문 (for · while)

같은 일을 여러 번 해야 할 때, 코드를 복사하지 않고 반복문으로 컴퓨터에게 "이만큼 반복해"라고 시킵니다.

  • for 문 — 정해진 횟수나 목록을 하나씩 돌 때. "1번부터 10번까지" 같은 경우.
  • while 문 — 끝나는 횟수는 모르지만 조건이 참인 동안 계속 반복할 때.
조건이 참인 동안 코드를 실행하고 다시 조건으로 — 거짓이 되면 빠져나감
loops.py
# for: 1부터 5까지 더하기
total = 0
for i in range(1, 6):     # i = 1, 2, 3, 4, 5
    total += i            # total = total + i
print(total)              # 15

# while: 입력이 "끝"일 때까지 반복
while True:
    word = input("단어(끝내려면 '끝'): ")
    if word == "끝":
        break             # 반복문 즉시 탈출
    print("입력한 단어:", word)
🚨 무한 루프 주의

while 문의 조건이 영원히 참이면 프로그램이 멈추지 않습니다. 언젠가 조건이 거짓이 되도록(또는 break로 탈출하도록) 반드시 설계하세요.

함수

함수(function)는 자주 쓰는 코드 묶음에 이름을 붙여, 필요할 때마다 그 이름만 불러 쓰는 기능입니다. "재료를 넣으면 결과가 나오는 기계"라고 생각하면 됩니다. 같은 코드를 반복해 쓰지 않게 해주고, 프로그램을 작은 단위로 나눠 이해하기 쉽게 만듭니다.

값을 넣으면(매개변수) 함수가 처리해 결과를 돌려줌(반환값)
functions.py
def add(a, b):            # a, b 는 매개변수(받는 값)
    result = a + b
    return result         # 계산 결과를 돌려줌

def greet(name):
    print(f"안녕하세요, {name}님!")

x = add(3, 4)             # 함수 호출 → x 에 7 이 담김
print(x)                  # 7
greet("민수")             # 안녕하세요, 민수님!
💡 함수를 나누는 기준

하나의 함수는 하나의 일만 하도록 만드는 게 좋습니다. 이름만 봐도 무슨 일을 하는지 알 수 있으면 잘 지은 함수입니다.

디버깅 기초

코드가 처음부터 완벽하게 동작하는 일은 거의 없습니다. 디버깅(debugging)은 프로그램의 오류(버그)를 찾아 고치는 과정으로, 프로그래밍의 절반을 차지하는 핵심 능력입니다.

오류는 크게 두 종류입니다.

오류 종류언제예시
문법 오류실행 전, 컴퓨터가 못 알아들을 때괄호 빠짐, 들여쓰기 틀림
논리 오류실행은 되지만 결과가 틀릴 때+ 를 - 로 잘못 씀

가장 쉽고 강력한 디버깅 방법은 중간 값을 print()로 찍어 보는 것입니다. 어디까지 제대로 왔는지, 변수에 무슨 값이 들었는지를 눈으로 확인하면 문제 지점이 보입니다.

debug.py
def average(numbers):
    total = sum(numbers)
    print("DEBUG total =", total)   # 중간 값 확인용
    print("DEBUG 개수 =", len(numbers))
    return total / len(numbers)

scores = [80, 90, 100]
print(average(scores))   # DEBUG 출력으로 계산 과정을 눈으로 추적
📌 오류 메시지는 친구다

빨간 오류 메시지는 무섭게 보이지만, 맨 아래 줄에 "무엇이 몇 번째 줄에서 잘못됐는지"가 적혀 있습니다. 겁내지 말고 끝줄부터 읽어 보세요.

문자열 다루기

문자열(string)은 글자들이 줄지어 늘어선 자료형입니다. 이름·주소·메시지처럼 사람이 읽는 정보는 거의 다 문자열입니다. 문자열은 각 글자마다 0부터 번호(인덱스)가 매겨져 있어, 원하는 부분만 잘라 쓰거나 이어 붙일 수 있습니다.

"Python" 의 각 글자엔 0부터 번호가 붙음 — s[0] = 'P', s[2] = 't'
strings.py
s = "Python"

print(s[0])        # 'P'    — 첫 글자(인덱스 0)
print(s[-1])       # 'n'    — 음수는 뒤에서부터
print(s[0:3])      # 'Pyt'  — 슬라이싱: 0~2번 글자
print(len(s))      # 6      — 글자 수

print(s.upper())       # 'PYTHON'  — 대문자로
print(s.replace("P", "J"))  # 'Jython'  — 바꾸기
print("3" + "4")       # '34'   — 문자열 + 는 이어 붙이기

name = "민수"
print(f"안녕, {name}님!")   # f-문자열: 변수를 글 속에 끼워넣기
💡 f-문자열을 쓰자

변수를 문장에 섞을 때는 f"...{변수}..." 형태의 f-문자열이 가장 읽기 쉽습니다. + 로 일일이 이어 붙이는 것보다 깔끔하고 실수도 적습니다.

컬렉션: 리스트와 딕셔너리

값 하나가 아니라 여러 값을 한 묶음으로 다뤄야 할 때 컬렉션을 씁니다. 가장 많이 쓰는 둘은 리스트와 딕셔너리입니다.

  • 리스트(list) — 값을 순서대로 줄 세운 묶음. [ ] 로 만들고 번호로 꺼냅니다. 장바구니 목록 같은 것.
  • 딕셔너리(dict) — 이름표(키) ↔ 값의 짝으로 저장. { } 로 만들고 키로 꺼냅니다. 단어장(단어→뜻) 같은 것.
리스트는 번호로, 딕셔너리는 키(이름)로 값을 꺼냄
collections.py
# 리스트 — 순서가 있는 묶음
fruits = ["사과", "배", "감"]
print(fruits[0])      # '사과'
fruits.append("귤")   # 맨 뒤에 추가
print(len(fruits))    # 4

for f in fruits:      # 하나씩 꺼내 반복
    print(f)

# 딕셔너리 — 키로 값을 꺼냄
prices = {"apple": 1500, "banana": 900}
print(prices["apple"])      # 1500
prices["grape"] = 2000      # 새 짝 추가
print("apple" in prices)    # True — 키 존재 확인

for name, price in prices.items():
    print(name, price)
💡 어떤 걸 쓸까

"몇 번째"가 중요하면 리스트, "이름으로 바로 찾기"가 중요하면 딕셔너리입니다. 이 둘은 2학년 자료구조에서 배열·해시 테이블로 다시 만나게 됩니다.

예외 처리 (try-except)

프로그램은 실행 도중 예상 못 한 일을 만납니다. 숫자를 0으로 나누거나, 사용자가 숫자 대신 글자를 입력하는 경우입니다. 이런 예외(exception)가 나면 프로그램은 그 자리에서 멈춰 버립니다. try-except는 "혹시 오류가 나면 멈추지 말고 이렇게 대처해"라고 미리 대비하는 안전망입니다.

try 안에서 오류가 나면 except로, 무사하면 그대로 진행 — 프로그램이 멈추지 않음
exception.py
try:
    num = int(input("숫자를 입력하세요: "))
    result = 100 / num
    print("결과:", result)
except ValueError:
    print("숫자가 아니에요! 다시 시도하세요.")   # 글자를 넣었을 때
except ZeroDivisionError:
    print("0으로는 나눌 수 없어요.")              # 0을 넣었을 때
finally:
    print("계산을 끝냅니다.")   # 오류가 나든 안 나든 항상 실행
⚠️ 모든 오류를 그냥 무시하진 말기

except: 만 덜렁 써서 모든 오류를 삼키면, 정작 진짜 버그를 놓칩니다. 어떤 오류를 잡을지(ValueError 등) 구체적으로 적는 습관이 좋습니다.

파일 입출력

지금까지 만든 값은 프로그램이 끝나면 사라졌습니다. 파일에 저장하면 데이터를 오래 남길 수 있습니다. 파이썬에서는 파일을 "열고 → 읽거나 쓰고 → 닫는" 순서로 다룹니다. with 구문을 쓰면 닫기를 자동으로 해 주어 안전합니다.

모드의미주의
"r"읽기(read)파일이 없으면 오류
"w"쓰기(write)기존 내용을 덮어씀
"a"이어쓰기(append)뒤에 덧붙임
files.py
# 파일에 쓰기 — with 가 끝나면 자동으로 닫힘
with open("memo.txt", "w", encoding="utf-8") as f:
    f.write("첫 줄입니다.\n")
    f.write("둘째 줄입니다.\n")

# 파일에서 읽기
with open("memo.txt", "r", encoding="utf-8") as f:
    for line in f:                # 한 줄씩 읽기
        print(line.strip())       # strip()으로 줄바꿈 제거
💡 encoding="utf-8"

한글이 깨지지 않게 하려면 파일을 열 때 encoding="utf-8"을 붙이는 습관을 들이세요. 운영체제에 따라 기본값이 달라 한글이 깨지는 일을 막아 줍니다.

모듈과 라이브러리 (import)

세상의 모든 코드를 직접 짤 필요는 없습니다. 누군가 미리 만들어 둔 코드 묶음(모듈·라이브러리)을 import 한 줄로 가져다 쓸 수 있습니다. 수학 계산, 난수, 날짜, 인터넷 통신까지 — "바퀴를 다시 발명하지 않는" 것이 프로그래밍의 핵심 습관입니다.

import 한 줄로 남이 만든 기능을 내 프로그램에 연결
modules.py
import math                  # 모듈 전체 가져오기
from random import randint    # 모듈에서 일부만 가져오기

print(math.sqrt(16))     # 4.0  — 제곱근
print(math.pi)           # 3.141592...
print(randint(1, 6))     # 1~6 사이 주사위 한 번

import datetime
today = datetime.date.today()
print(today)             # 오늘 날짜
📌 표준 라이브러리와 외부 라이브러리

파이썬에 처음부터 들어 있는 것을 표준 라이브러리(math, random 등)라 하고, pip install 로 따로 설치하는 것을 외부 라이브러리(numpy, requests 등)라 합니다.

주석과 좋은 코드 스타일

코드는 컴퓨터만 읽는 게 아니라 미래의 나와 동료도 읽습니다. 한 달 뒤의 나는 오늘 짠 코드를 기억하지 못합니다. 읽기 좋은 코드를 쓰는 것은 예의이자 실력입니다.

  • 주석(comment) — # 뒤의 글은 실행되지 않습니다. "왜 이렇게 했는지"를 적습니다(무엇을 하는지는 코드가 이미 말해 줍니다).
  • 의미 있는 이름 — a, tmp 보다 total_price, user_age.
  • 한 함수는 한 가지 일 — 길어지면 작은 함수로 쪼갭니다.
style.py
# ❌ 나쁜 예: 이름만 봐선 무슨 값인지 모름
def f(x, y):
    return x * y * 0.9   # 0.9가 뭐지?

문제를 작은 단계로 쪼개기

큰 문제 앞에서 막막한 건 당연합니다. 프로그래밍 실력의 핵심은 코드 암기가 아니라, 큰 문제를 컴퓨터가 한 번에 할 수 있는 작은 단계들로 나누는 사고(알고리즘적 사고)입니다. 이것만 되면 코드는 한 줄씩 채우기만 하면 됩니다.

예를 들어 "성적표에서 합격자 명단 뽑기"는 이렇게 쪼갤 수 있습니다.

하나의 큰 문제를 차례대로 풀 수 있는 작은 단계 셋으로 분해
decompose.py
# 큰 문제를 위에서 나눈 3단계 그대로 코드로 옮기기
students = [("민수", 80), ("영희", 45), ("철수", 92)]

# 2. 60점 이상만 거르고  3. 이름만 모으기
passed = []
for name, score in students:        # 1. 한 명씩 읽기
    if score >= 60:                 # 2. 합격 조건
        passed.append(name)         # 3. 명단에 추가

print("합격자:", passed)   # 합격자: ['민수', '철수']
📌 막히면 "더 작게"

한 단계가 여전히 어렵다면, 그 단계를 다시 더 작게 쪼개세요. 종이에 한국어로 순서를 먼저 적고(의사코드), 그 다음 한 줄씩 코드로 바꾸면 훨씬 쉽습니다.

🎯 핵심 정리
  • 프로그램은 입력 → 처리 → 출력의 흐름으로, 위에서 아래로 한 줄씩 실행된다.
  • 변수는 값을 담는 이름표 붙은 상자이고, 자료형은 그 값의 종류다.
  • 조건문(if)은 상황에 따라 길을 나누고, 반복문(for·while)은 같은 일을 되풀이한다.
  • 함수는 코드 묶음에 이름을 붙여 재사용하게 해 준다.
  • 디버깅은 print로 중간 값을 확인하고 오류 메시지를 읽는 데서 시작한다.
  • 문자열은 인덱스·슬라이싱으로 자르고, 리스트는 순서로·딕셔너리는 키로 여러 값을 묶는다.
  • try-except로 오류에 대비하고, 파일 입출력으로 데이터를 오래 저장한다.
  • import로 남이 만든 라이브러리를 가져다 쓰고, 주석과 좋은 이름으로 읽기 쉬운 코드를 짠다.
  • 실력의 핵심은 큰 문제를 작은 단계로 쪼개는 알고리즘적 사고다.
변수(Variable)
값을 담아 두고 이름으로 꺼내 쓰는 저장 공간.
자료형(Data Type)
정수·실수·문자열·불처럼 값의 종류를 구분하는 분류.
조건문(if)
참/거짓에 따라 실행할 코드를 갈라 주는 문장.
반복문(Loop)
같은 코드를 여러 번 실행하게 하는 for·while 문.
함수(Function)
이름 붙인 코드 묶음. 값을 받아 처리하고 결과를 돌려줌.
디버깅(Debugging)
프로그램의 오류(버그)를 찾아 고치는 과정.
문자열(String)
글자들이 순서대로 이어진 자료형. 인덱스·슬라이싱으로 다룸.
리스트(List)
여러 값을 순서대로 담는 컬렉션. 번호(인덱스)로 접근.
딕셔너리(Dict)
키와 값의 짝으로 저장하는 컬렉션. 키로 값을 꺼냄.
예외(Exception)
실행 중 생기는 오류. try-except로 미리 대비함.
모듈/라이브러리
미리 만들어진 코드 묶음. import로 가져다 씀.
주석(Comment)
# 뒤에 적는, 실행되지 않는 설명문.
문제 분해(Decomposition)
큰 문제를 풀 수 있는 작은 단계들로 나누는 사고.
