자료구조

데이터를 어떻게 담느냐가 프로그램의 속도와 메모리를 결정합니다. 상황에 맞는 그릇을 고르는 법을 배웁니다.

자료구조란?

자료구조(data structure)는 데이터를 저장하고 정리하는 방식입니다. 같은 데이터라도 어떤 구조에 담느냐에 따라, "특정 값 찾기"나 "맨 앞에 끼워넣기" 같은 작업이 빠를 수도, 느릴 수도 있습니다.

요리에 비유하면 데이터는 재료, 자료구조는 그것을 담는 그릇입니다. 국을 접시에, 샐러드를 컵에 담으면 불편하듯, 작업에 맞지 않는 자료구조를 쓰면 프로그램이 느려집니다.

📌 이 페이지에서 배우는 것

배열 · 연결 리스트 · 스택 · 큐 · 트리 · 해시 테이블, 그리고 각 구조의 성능을 비교하는 시간 복잡도까지 다룹니다.

먼저: 시간 복잡도(Big-O)

자료구조를 비교하려면 "데이터가 많아질 때 작업이 얼마나 느려지는가"를 표현하는 공통 언어가 필요합니다. 그게 빅오 표기법입니다. 입력 크기 n이 커질 때의 증가 추세만 봅니다.

아래로 갈수록 좋음: O(1) < O(log n) < O(n) < O(n²)
표기이름예시
O(1)상수배열에서 인덱스로 접근
O(log n)로그이진 탐색, 균형 트리 탐색
O(n)선형리스트 전체 훑기
O(n²)제곱이중 반복문(버블 정렬)

배열 (Array)

배열은 같은 크기의 칸을 메모리에 연속으로 늘어놓은 구조입니다. 각 칸에는 0부터 시작하는 번호(인덱스)가 있어, 인덱스만 알면 곧바로 그 칸에 접근할 수 있습니다 — 그래서 읽기가 O(1)로 매우 빠릅니다.

인덱스로 어느 칸이든 한 번에 접근 — arr[3] = 40
array.py
arr = [10, 20, 30, 40, 50]

print(arr[3])      # 40  — 인덱스 접근: O(1)
arr.append(60)     # 맨 뒤 추가: 평균 O(1)
arr.insert(0, 5)   # 맨 앞 삽입: 뒤 원소를 모두 밀어야 함 → O(n)
⚠️ 약점

중간/앞에 삽입·삭제하면 뒤의 원소들을 한 칸씩 옮겨야 해서 O(n)입니다. 잦은 삽입·삭제가 필요하면 연결 리스트를 고려하세요.

연결 리스트 (Linked List)

연결 리스트는 데이터를 담은 노드(node)들이 "다음 노드 주소"를 서로 가리키며 사슬처럼 이어진 구조입니다. 메모리에 흩어져 있어도 됩니다. 그래서 중간 삽입·삭제가 O(1)(연결만 바꾸면 됨)로 빠르지만, n번째 원소를 찾으려면 앞에서부터 따라가야 해 접근이 O(n)입니다.

각 노드가 [값 | 다음 노드 주소] 를 들고 있음
linked_list.py
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None        # 다음 노드를 가리킴

# 10 -> 20 -> 30
head = Node(10)
head.next = Node(20)
head.next.next = Node(30)

# head 와 head.next 사이에 15 삽입 — 연결만 바꾸면 끝(O(1))
node = Node(15)
node.next = head.next
head.next = node

스택과 큐 (Stack & Queue)

스택과 큐는 "넣고 빼는 순서"에 규칙을 둔 구조입니다.

  • 스택(Stack) — LIFO(Last In, First Out). 마지막에 넣은 게 먼저 나옴. 접시 쌓기, 되돌리기(Undo), 함수 호출 스택.
  • 큐(Queue) — FIFO(First In, First Out). 먼저 넣은 게 먼저 나옴. 줄 서기, 작업 대기열, 너비 우선 탐색(BFS).
스택은 한쪽 끝에서만, 큐는 양쪽 끝에서 출입
stack_queue.py
from collections import deque

stack = []
stack.append("A"); stack.append("B")
print(stack.pop())     # "B"  — 마지막에 넣은 것이 먼저

queue = deque()
queue.append("A"); queue.append("B")
print(queue.popleft()) # "A"  — 먼저 넣은 것이 먼저
💡 팁

파이썬에서 큐는 list.pop(0) 대신 collections.deque를 쓰세요. 리스트의 앞쪽 제거는 O(n)이지만 deque는 O(1)입니다.

트리 (Tree)

트리는 데이터가 부모-자식 관계로 가지치는 계층 구조입니다. 맨 위를 루트(root), 끝을 리프(leaf)라 합니다. 폴더 구조, HTML DOM, 조직도가 모두 트리입니다.

특히 이진 탐색 트리(BST)는 "왼쪽 자식 < 부모 < 오른쪽 자식" 규칙을 지켜, 탐색할 때마다 후보를 절반으로 줄여 O(log n)에 값을 찾습니다.

BST: 왼쪽은 작고, 오른쪽은 큰 값 → 탐색 시 절반씩 제거
bst.py
class TreeNode:
    def __init__(self, key):
        self.key = key
        self.left = self.right = None

def search(node, target):
    if node is None or node.key == target:
        return node
    if target < node.key:
        return search(node.left, target)   # 왼쪽만 탐색
    return search(node.right, target)      # 오른쪽만 탐색

해시 테이블 (Hash Table)

해시 테이블은 키를 해시 함수에 넣어 나온 숫자를 배열 인덱스로 삼아 값을 저장합니다. 덕분에 키만 알면 평균 O(1)에 값을 넣고 찾습니다. 파이썬의 dict, 자바스크립트의 Map/Object가 바로 해시 테이블입니다.

키 → 해시 함수 → 배열 위치. 키로 곧장 값에 도달
hash_table.py
prices = {}              # dict = 해시 테이블
prices["apple"] = 1500   # 삽입: 평균 O(1)
prices["banana"] = 900

print(prices["apple"])   # 조회: 평균 O(1) → 1500
print("grape" in prices) # 존재 확인도 O(1) → False
⚠️ 충돌(collision)

서로 다른 키가 같은 인덱스로 가는 경우가 생깁니다(충돌). 체이닝·개방 주소법으로 해결하며, 충돌이 많아지면 최악엔 O(n)까지 느려질 수 있습니다.

한눈에 비교

자료구조접근탐색삽입/삭제잘 맞는 상황
배열O(1)O(n)O(n)인덱스 접근이 잦을 때
연결 리스트O(n)O(n)O(1)삽입·삭제가 잦을 때
스택/큐——O(1)순서 규칙이 필요할 때
이진 탐색 트리O(log n)O(log n)O(log n)정렬된 상태로 탐색
해시 테이블—O(1)O(1)키-값 빠른 조회

정답인 자료구조는 없습니다. 가장 자주 하는 작업이 무엇인지로 고르세요.

덱 (Deque)

덱(deque, double-ended queue)은 "양쪽 끝 큐"입니다. 스택은 한쪽 끝, 큐는 들어가고 나가는 끝이 정해져 있지만, 덱은 앞뒤 양쪽에서 모두 O(1)로 넣고 뺄 수 있습니다. 그래서 스택으로도, 큐로도 쓸 수 있는 만능 그릇입니다.

양쪽 끝 모두에서 O(1)로 출입 — 스택+큐의 결합
deque.py
from collections import deque

dq = deque(["B"])
dq.appendleft("A")    # 앞에 넣기: O(1)  →  [A, B]
dq.append("C")        # 뒤에 넣기: O(1)  →  [A, B, C]
print(dq.popleft())   # 앞에서 빼기: "A"
print(dq.pop())       # 뒤에서 빼기: "C"
💡 어디에 쓰나

최근 항목 N개만 유지하는 슬라이딩 윈도, 양쪽에서 처리하는 회문 검사, 그래프의 BFS 대기열에 자주 쓰입니다. deque(maxlen=N)으로 길이 제한도 줄 수 있습니다.

우선순위 큐와 힙 (Priority Queue & Heap)

보통 큐는 "먼저 온 순서"로 나오지만, 우선순위 큐는 가장 우선순위가 높은(보통 가장 작은) 값이 먼저 나옵니다. 이를 빠르게 구현하는 자료구조가 힙(heap)입니다. 힙은 "부모가 항상 자식보다 작다(최소 힙)"는 규칙을 지키는 완전 이진 트리로, 최솟값 꺼내기와 삽입을 모두 O(log n)에 합니다.

최소 힙: 부모 ≤ 자식. 가장 작은 값은 항상 꼭대기에
heap.py
import heapq

h = []
heapq.heappush(h, 5)     # 삽입: O(log n)
heapq.heappush(h, 1)
heapq.heappush(h, 3)

print(heapq.heappop(h))  # 1  — 항상 최솟값부터: O(log n)
print(heapq.heappop(h))  # 3

# (우선순위, 데이터) 튜플로 우선순위 큐 활용
tasks = []
heapq.heappush(tasks, (2, "낮은 우선"))
heapq.heappush(tasks, (1, "급한 일"))
print(heapq.heappop(tasks))   # (1, '급한 일')
⚠️ 정렬과는 다르다

힙은 전체가 정렬돼 있지 않습니다. 오직 최솟값(루트)만 보장합니다. 그래서 "매번 최솟값/최댓값만 필요"할 때 정렬보다 효율적입니다. 다익스트라 최단 경로, 작업 스케줄러의 핵심 부품입니다.

그래프 표현 (인접 행렬·인접 리스트)

그래프는 정점(node)들이 간선(edge)으로 연결된 구조로, 지도·SNS 친구 관계·도로망을 나타냅니다. 컴퓨터에 그래프를 담는 방법은 크게 두 가지입니다. 인접 행렬은 2차원 표로, 인접 리스트는 각 정점의 이웃 목록으로 표현합니다.

같은 그래프, 두 가지 저장 방식
graph.py
# 인접 행렬: matrix[i][j]==1 이면 i-j 연결
matrix = [
    [0, 1, 1],
    [1, 0, 0],
    [1, 0, 0],
]
print(matrix[0][1])     # 1  — 0과 1은 연결됨 (확인 O(1))

# 인접 리스트: 각 정점의 이웃만 저장 (공간 절약)
adj = {0: [1, 2], 1: [0], 2: [0]}
print(adj[0])           # [1, 2]  — 0의 이웃들
방식공간연결 확인유리한 경우
인접 행렬O(V²)O(1)간선이 빽빽한 그래프
인접 리스트O(V+E)O(이웃 수)간선이 듬성한 그래프
💡 보통은 인접 리스트

현실의 그래프(SNS, 도로망)는 대부분 간선이 듬성합니다. 그래서 공간을 아끼는 인접 리스트를 기본으로 쓰고, 정점 수가 작고 간선이 많을 때만 인접 행렬을 고려합니다.

균형 트리 개념 (AVL·레드블랙)

이진 탐색 트리는 O(log n)이 장점이지만, 한 가지 함정이 있습니다. 값을 정렬된 순서로 넣으면 트리가 한쪽으로 쏠려 연결 리스트처럼 변해 탐색이 O(n)으로 추락합니다. 이를 막으려고 삽입·삭제할 때마다 스스로 균형을 맞추는 트리가 균형 트리입니다.

같은 값이라도 균형을 맞추면 높이가 낮아져 탐색이 빨라진다
종류균형 방식특징
AVL 트리높이 차를 1 이하로 엄격히탐색이 가장 빠름(엄격)
레드블랙 트리노드에 색을 칠해 느슨하게삽입·삭제가 더 빠름(실무 표준)
📌 이미 쓰고 있다

입문 단계에서 직접 구현할 필요는 없습니다. 다만 자바의 TreeMap·C++의 std::map이 레드블랙 트리로 만들어졌다는 점만 기억하세요. "정렬된 상태를 유지하며 O(log n)"이 필요할 때 내부에서 균형 트리가 일합니다.

트라이 (Trie)

트라이(trie)는 문자열을 글자 단위로 가지치며 저장하는 트리입니다. 공통된 앞부분(접두사)을 한 번만 저장하므로, "특정 접두사로 시작하는 단어 찾기"가 매우 빠릅니다. 검색창 자동완성, 사전, 전화번호부의 핵심 자료구조입니다.

"cat"과 "cow"가 첫 글자 c를 공유 — 초록은 단어 끝
trie.py
class Trie:
    def __init__(self):
        self.root = {}

    def insert(self, word):
        node = self.root
        for ch in word:                  # 글자 단위로 내려감
            node = node.setdefault(ch, {})
        node["$"] = True                 # 단어 끝 표시

    def starts_with(self, prefix):       # 접두사 존재 확인
        node = self.root
        for ch in prefix:
            if ch not in node: return False
            node = node[ch]
        return True

t = Trie()
t.insert("cat"); t.insert("cow")
print(t.starts_with("ca"))   # True
print(t.starts_with("dog"))  # False
💡 강점과 비용

길이 L짜리 단어를 찾는 데 단어 개수와 무관하게 O(L)입니다. 대신 글자마다 노드를 만들어 메모리를 많이 씁니다. 접두사 검색이 핵심일 때 진가를 발휘합니다.

유니온-파인드 (분리 집합)

유니온-파인드(union-find)는 원소들을 겹치지 않는 그룹(집합)으로 묶고, "이 둘이 같은 그룹인가?"를 빠르게 답하는 자료구조입니다. 친구의 친구를 같은 무리로 합치거나, 네트워크가 모두 연결됐는지 확인할 때 씁니다. 핵심 연산은 두 그룹을 합치는 union과, 대표를 찾는 find입니다.

union_find.py
parent = list(range(5))      # 처음엔 각자 자기 자신이 대표

def find(x):                 # x가 속한 그룹의 대표 찾기
    while parent[x] != x:
        parent[x] = parent[parent[x]]   # 경로 압축(속도 ↑)
        x = parent[x]
    return x

def union(a, b):             # 두 그룹 합치기
    parent[find(a)] = find(b)

union(0, 1)                  # 0과 1을 같은 그룹으로
union(1, 2)                  # 0,1,2 가 한 그룹
print(find(0) == find(2))    # True  — 같은 그룹
print(find(0) == find(3))    # False — 다른 그룹
⚠️ 거의 O(1)

경로 압축과 랭크(크기) 기준 합치기를 함께 쓰면, 한 번의 연산이 사실상 상수 시간에 가깝게 동작합니다. 크루스칼 최소 신장 트리, 사이클 판별의 단골 부품입니다.

집합 (Set)

집합(set)은 중복 없는 원소들의 모음입니다. 순서가 없고 같은 값을 두 번 담지 않습니다. 내부적으로 해시 테이블로 구현돼, "이 값이 있는가?"라는 멤버십 확인이 평균 O(1)로 매우 빠릅니다. 또 합집합·교집합·차집합 같은 수학 연산을 그대로 지원합니다.

set.py
a = {1, 2, 3, 3}     # 중복 자동 제거 → {1, 2, 3}
print(2 in a)        # True  — 포함 확인: 평균 O(1)

b = {3, 4, 5}
print(a | b)         # 합집합  {1, 2, 3, 4, 5}
print(a & b)         # 교집합  {3}
print(a - b)         # 차집합  {1, 2}

# 리스트 중복 제거에 흔히 사용
nums = [1, 1, 2, 2, 3]
print(list(set(nums)))   # [1, 2, 3]
연산기호평균 복잡도
포함 확인x in sO(1)
추가/삭제add/removeO(1)
합·교·차집합| & -O(두 집합 크기)
💡 리스트 대신 집합

"이미 본 값인가?"를 자주 확인한다면 리스트(x in list = O(n)) 대신 집합(O(1))을 쓰세요. 방문 기록, 중복 제거, 빠른 필터링에서 성능 차이가 큽니다.

🎯 핵심 정리
  • 자료구조 선택의 기준은 "어떤 작업을 가장 자주 하는가"이다.
  • 배열은 접근이 빠르고(O(1)), 연결 리스트는 삽입·삭제가 빠르다(O(1)).
  • 스택은 LIFO, 큐는 FIFO — 순서 규칙이 핵심.
  • 이진 탐색 트리는 절반씩 줄이며 O(log n)에 탐색한다.
  • 해시 테이블은 키로 평균 O(1)에 값을 찾는, 가장 많이 쓰는 구조다.
  • 덱은 양쪽 끝 모두에서 O(1)로 넣고 빼는 만능 큐다.
  • 힙은 최솟값(또는 최댓값)을 O(log n)에 꺼내는 우선순위 큐의 핵심이다.
  • 그래프는 인접 행렬(빽빽할 때)이나 인접 리스트(듬성할 때)로 표현한다.
  • 균형 트리(AVL·레드블랙)는 쏠림을 막아 O(log n)을 보장한다.
  • 트라이는 접두사를 공유해 자동완성·사전 검색에 강하다.
  • 유니온-파인드는 그룹 합치기와 같은 그룹 판별을 거의 O(1)에 한다.
  • 집합은 중복 없는 모음으로, 포함 확인과 집합 연산이 빠르다.
노드(Node)
값과 연결 정보를 담은 자료구조의 기본 단위.
빅오(Big-O)
입력이 커질 때 작업량이 증가하는 추세를 나타내는 표기법.
LIFO / FIFO
각각 후입선출(스택), 선입선출(큐) 규칙.
해시 함수
임의의 키를 고정 범위의 숫자(인덱스)로 변환하는 함수.
덱(Deque)
앞뒤 양쪽 끝에서 모두 O(1)로 넣고 뺄 수 있는 큐.
힙(Heap)
부모가 자식보다 작거나 큰 규칙을 지켜 최솟/최댓값을 빨리 꺼내는 트리.
우선순위 큐
우선순위가 높은 원소가 먼저 나오는 큐(보통 힙으로 구현).
인접 행렬 / 인접 리스트
그래프를 각각 2차원 표 / 이웃 목록으로 저장하는 방식.
균형 트리
삽입·삭제 시 스스로 균형을 맞춰 O(log n)을 유지하는 트리(AVL·레드블랙).
트라이(Trie)
문자열을 글자 단위로 가지치며 접두사를 공유해 저장하는 트리.
유니온-파인드
원소를 겹치지 않는 집합으로 묶고 같은 집합 여부를 빠르게 판별.
집합(Set)
중복이 없고 순서가 없는 원소들의 모음. 포함 확인이 평균 O(1).
