컴퓨터 구조

우리가 짠 코드는 결국 0과 1이 되어 전기로 흐릅니다. 컴퓨터가 명령을 받아 계산하기까지, 그 속을 열어 봅니다.

컴퓨터 구조 개요

컴퓨터 구조(computer architecture)는 컴퓨터가 내부에서 어떻게 동작하는지를 다루는 분야입니다. 프로그래밍이 "무엇을 시킬지"라면, 컴퓨터 구조는 "그 명령이 하드웨어에서 어떻게 실행되는지"입니다.

가장 기본 골격은 폰 노이만 구조입니다. ① 계산하는 CPU, ② 데이터와 명령을 담는 메모리, ③ 바깥과 소통하는 입출력 장치가 하나의 통로(버스)로 연결됩니다.

CPU·메모리·입출력이 버스로 연결된 폰 노이만 구조
📌 이 페이지에서 배우는 것

이진수와 2의 보수 → 논리 게이트 → CPU 구성(ALU·레지스터·제어장치) → 명령어 실행 사이클 → 메모리 계층 → 파이프라이닝까지, 하드웨어가 코드를 실행하는 흐름을 따라갑니다.

이진수와 2의 보수

컴퓨터는 전기가 흐르거나(1) 흐르지 않는(0) 두 상태만 구분합니다. 그래서 모든 숫자를 이진수(binary)로 표현합니다. 각 자리는 2의 거듭제곱이며, 한 자리를 비트(bit), 8비트 묶음을 바이트(byte)라 합니다.

이진수 1011 = 8 + 0 + 2 + 1 = 십진수 11

음수는 어떻게 표현할까요? 가장 널리 쓰는 방식이 2의 보수(two's complement)입니다. 모든 비트를 뒤집고(0↔1) 1을 더하면 음수가 됩니다. 이 방식이면 뺄셈을 덧셈 회로로 그대로 처리할 수 있어 하드웨어가 단순해집니다.

two_complement.c
// 8비트 기준: 5를 -5로 (2의 보수)
//   5 = 0000 0101
//   비트 반전     1111 1010
//   +1            1111 1011  → 이게 -5

#include <stdio.h>
int main() {
    char a = 5;          // 0000 0101
    char b = -5;         // 1111 1011 (2의 보수)
    printf("%d\n", a + b);   // 0 — 덧셈 회로로 뺄셈 처리
    return 0;
}
💡 왜 2의 보수인가

0이 하나뿐이고(+0/-0 구분 없음), 덧셈·뺄셈을 같은 회로로 처리할 수 있습니다. 그래서 거의 모든 컴퓨터가 정수 음수를 2의 보수로 저장합니다.

논리 게이트

모든 계산은 결국 논리 게이트(logic gate)라는 작은 스위치들의 조합입니다. 게이트는 0/1 입력을 받아 0/1을 출력합니다. 기본은 AND(둘 다 1일 때만 1), OR(하나라도 1이면 1), NOT(뒤집기)입니다.

기본 게이트 — 이 작은 스위치들을 수억 개 모으면 CPU가 된다
ABA AND BA OR BNOT A
00001
01011
10010
11110

이 게이트들을 조합하면 덧셈을 하는 가산기(adder), 값을 기억하는 회로 등 더 큰 부품을 만들 수 있습니다. 컴퓨터의 모든 계산은 이 단순한 논리 연산의 거대한 누적입니다.

CPU 구성

CPU(중앙처리장치)는 계산을 담당하는 컴퓨터의 두뇌입니다. 크게 세 부분으로 나뉩니다.

  • ALU(산술논리장치) — 실제 덧셈·뺄셈·AND·OR 같은 연산을 수행.
  • 레지스터(register) — CPU 안의 아주 빠른 초소형 저장 공간. 지금 다루는 값을 잠깐 둠.
  • 제어장치(Control Unit) — 명령어를 해석해 "이번엔 ALU에 덧셈을 시켜라" 같은 신호를 내보냄.
CPU = ALU(계산) + 레지스터(임시 저장) + 제어장치(지휘)
add.asm
; C 코드:  c = a + b;  를 어셈블리로 본 모습
mov  eax, [a]    ; 메모리의 a 값을 레지스터 eax 로
add  eax, [b]    ; eax 에 b 를 더함  ← ALU가 수행
mov  [c], eax    ; 결과를 메모리 c 에 저장

명령어 실행 사이클

CPU는 한 명령을 처리할 때 늘 같은 세 단계를 반복합니다. 이를 fetch-decode-execute 사이클이라 합니다.

  1. Fetch(인출) — 메모리에서 다음 명령어를 가져옴. (현재 위치는 PC 레지스터가 가리킴)
  2. Decode(해석) — 제어장치가 "이게 무슨 명령인지" 해독함.
  3. Execute(실행) — ALU 등이 실제 연산을 수행하고 결과를 저장함.
이 사이클이 초당 수십억 번 돌면서 프로그램이 실행된다
💡 클럭(clock)

이 사이클의 박자를 맞추는 것이 클럭입니다. 3GHz CPU는 1초에 30억 번의 클럭 신호를 내보냅니다 — 그만큼 빠르게 명령을 처리한다는 뜻입니다.

메모리 계층

빠른 저장 장치는 비싸고 작으며, 느린 장치는 싸고 큽니다. 그래서 컴퓨터는 여러 저장 장치를 계층(memory hierarchy)으로 쌓습니다. 위로 갈수록 빠르고 작고, 아래로 갈수록 느리고 큽니다.

자주 쓰는 데이터일수록 위쪽(빠른 곳)에 둔다

핵심은 캐시(cache)입니다. CPU가 방금 쓴 데이터는 곧 또 쓸 가능성이 높습니다(지역성). 그래서 RAM보다 훨씬 빠른 캐시에 복사해 두고, 다음에 같은 데이터가 필요하면 RAM까지 가지 않고 캐시에서 바로 꺼냅니다.

장치상대 속도대략 용량
레지스터가장 빠름수십~수백 바이트
캐시매우 빠름수 MB
RAM빠름수~수십 GB
SSD/HDD느림수백 GB~수 TB
⚠️ 캐시 미스(cache miss)

필요한 데이터가 캐시에 없으면 느린 RAM까지 가야 합니다. 이를 캐시 미스라 하며, 자주 발생하면 프로그램이 눈에 띄게 느려집니다. 데이터를 연속적으로 접근하면 미스가 줄어듭니다.

파이프라이닝

명령어 하나가 fetch→decode→execute를 모두 마칠 때까지 기다렸다 다음 명령을 시작하면 낭비가 큽니다. 파이프라이닝(pipelining)은 공장의 컨베이어 벨트처럼, 한 명령이 decode 단계에 있을 때 다음 명령은 이미 fetch를 시작하게 해 여러 명령을 겹쳐 처리합니다.

파이프라이닝: 명령들을 겹쳐 흘려보내 같은 시간에 더 많이 처리
⚠️ 해저드(hazard)

다음 명령이 앞 명령의 결과를 필요로 하거나, 분기(if)로 어디로 갈지 모를 때 파이프라인이 멈출 수 있습니다. 이를 해저드라 하며, CPU는 분기 예측 등으로 이를 줄이려 노력합니다.

부동소수점 표현 (IEEE 754)

정수는 2의 보수로 깔끔하게 표현되지만, 3.14 같은 실수는 어떻게 0과 1로 담을까요? 거의 모든 컴퓨터가 IEEE 754 표준을 씁니다. 과학적 표기법(예: 1.01 × 2³)처럼, 실수를 부호 · 지수 · 가수(尾數) 세 부분으로 쪼개 저장합니다.

32비트 단정밀도(float)는 부호 1비트, 지수 8비트, 가수 23비트로 나뉩니다. 값은 대략 (-1)부호 × 1.가수 × 2(지수-127)로 복원됩니다. 지수에 127을 빼는(바이어스) 이유는 음의 지수도 표현하기 위함입니다.

π ≈ 3.14를 IEEE 754 float으로: 부호(0) · 지수(1) · 가수로 분해
float_repr.py
import struct

x = 0.1 + 0.2
print(x)              # 0.30000000000000004  ← 정확히 0.3이 아니다!
print(x == 0.3)       # False

# float을 32비트 비트열로 직접 보기
bits = struct.pack(">f", 3.14)
print("".join(f"{b:08b}" for b in bits))
# 01000000010010001111010111000011  (부호1 지수8 가수23)
⚠️ 0.1 + 0.2 ≠ 0.3

0.1은 2진수로 무한 반복 소수라 정확히 저장할 수 없습니다. 그래서 부동소수점 계산엔 미세한 오차가 따릅니다. 돈 계산처럼 정확도가 중요하면 정수(센트 단위)나 십진 타입(decimal)을 쓰세요. 실수는 abs(a-b) < 1e-9처럼 비교합니다.

캐시 동작과 지역성

앞서 캐시가 빠른 임시 메모리라고 했습니다. 그 효과의 비밀은 지역성(locality)입니다. 프로그램은 ① 방금 쓴 데이터를 곧 또 쓰고(시간 지역성), ② 쓴 데이터의 바로 옆을 이어서 쓰는(공간 지역성) 경향이 있습니다. 캐시는 이 경향에 베팅합니다.

그래서 캐시는 한 바이트가 아니라 캐시 라인(보통 64바이트) 단위로 통째로 가져옵니다. 배열을 순서대로 훑으면 첫 접근(미스) 한 번에 옆 데이터까지 캐시에 올라와, 다음 수십 번은 빠른 적중(hit)이 됩니다.

미스 한 번에 64바이트가 통째로 올라오고, 옆 데이터는 적중으로 빠르게 읽힌다
locality.c
// 같은 2차원 배열을 두 순서로 합산 — 결과는 같지만 속도가 다르다
int sum = 0;
// (A) 행 우선: 메모리에 연속 접근 → 캐시 적중 多, 빠름
for (int i = 0; i < N; i++)
    for (int j = 0; j < N; j++)
        sum += a[i][j];

// (B) 열 우선: 멀리 건너뛰며 접근 → 캐시 미스 多, 느림
for (int j = 0; j < N; j++)
    for (int i = 0; i < N; i++)
        sum += a[i][j];   // (A)보다 수 배 느릴 수 있다
💡 적중률(hit rate)

적중률 = 적중 횟수 / 전체 접근. 평균 접근 시간 ≈ (적중률 × 캐시 속도) + (미스율 × RAM 속도)입니다. 미스 한 번이 적중의 수십~수백 배 느리므로, 적중률을 95%에서 99%로만 올려도 체감 속도가 크게 달라집니다. 데이터를 연속으로 배치하는 것만으로 공짜 성능을 얻습니다.

가상 메모리와 TLB

프로그램은 자기가 메모리를 통째로 혼자 쓰는 것처럼 동작합니다. 하지만 실제로는 여러 프로그램이 RAM을 나눠 씁니다. 이 착각을 만들어 주는 것이 가상 메모리(virtual memory)입니다. 프로그램이 쓰는 가상 주소를 하드웨어가 실제 물리 주소로 바꿔 줍니다.

변환표를 페이지 테이블이라 하는데, 이를 매번 RAM에서 찾으면 느립니다. 그래서 CPU 안에 최근 변환 결과를 캐싱하는 작은 하드웨어 TLB(Translation Lookaside Buffer)를 둡니다. TLB에 있으면(적중) 즉시 변환되고, 없으면(미스) 페이지 테이블을 찾아 채웁니다.

TLB에 변환이 있으면 즉시, 없으면 느린 페이지 테이블을 조회해 채운다
💡 페이지와 페이지 폴트

메모리는 보통 4KB 페이지 단위로 관리됩니다. 필요한 페이지가 RAM에 없고 디스크에 있으면 페이지 폴트가 발생해, OS가 디스크에서 RAM으로 불러옵니다. 가상 메모리 덕분에 RAM보다 큰 프로그램도 실행할 수 있고, 프로그램들이 서로의 메모리를 침범하지 못하게 격리됩니다.

입출력과 인터럽트

키보드 입력이나 디스크 읽기는 CPU 속도에 비하면 까마득히 느립니다. CPU가 "데이터 왔니?"를 계속 물어보며 기다리면(폴링) 그동안 아무 일도 못 합니다. 그래서 현대 컴퓨터는 인터럽트(interrupt)를 씁니다.

CPU는 입출력을 시켜 놓고 다른 일을 합니다. 장치가 일을 끝내면 CPU에 "나 끝났어!" 하고 신호(인터럽트)를 보냅니다. CPU는 하던 일을 잠시 멈추고, 그 상태를 저장한 뒤 인터럽트 핸들러를 실행하고, 끝나면 멈췄던 곳으로 정확히 돌아옵니다.

인터럽트: 장치가 부를 때만 CPU가 잠깐 대응하고 곧 본래 일로 복귀
방식동작CPU 효율
폴링(Polling)CPU가 계속 물어봄낮음 (계속 대기)
인터럽트장치가 끝나면 알림높음 (그동안 다른 일)
DMA전용 장치가 메모리로 직접 전송가장 높음 (CPU 거의 무관여)
💡 DMA(직접 메모리 접근)

대용량 전송(예: 디스크→RAM)을 CPU가 한 바이트씩 옮기면 비효율적입니다. DMA 컨트롤러가 CPU를 거치지 않고 장치와 메모리 사이를 직접 옮긴 뒤, 다 끝나면 인터럽트 한 번으로 알립니다. CPU는 그동안 계산에 전념합니다.

병렬성 (SIMD · 멀티코어)

클럭 속도를 무한히 올릴 수는 없습니다(발열·전력 한계). 그래서 현대 CPU는 동시에 여러 일을 하는 방향으로 발전했습니다. 크게 두 갈래입니다.

  • SIMD(Single Instruction, Multiple Data) — 명령 하나로 여러 데이터를 한꺼번에 처리. 예: 4개의 숫자를 한 번의 덧셈 명령으로 동시에 더함. 영상·과학 계산에 강력.
  • 멀티코어 — CPU 안에 독립된 코어를 여러 개 두어, 서로 다른 작업(스레드)을 진짜 동시에 실행.
SIMD는 데이터 병렬, 멀티코어는 작업 병렬 — 둘 다 "동시에 더 많이"
⚠️ 암달의 법칙

코어를 8개로 늘려도 8배 빨라지지 않습니다. 프로그램에는 순차적으로만 처리되는 부분이 남아 있기 때문입니다. 전체의 90%를 병렬화해도 나머지 10%가 발목을 잡아, 코어를 아무리 늘려도 최대 10배까지만 빨라집니다(암달의 법칙).

명령어 집합 (RISC vs CISC)

CPU가 이해하는 명령어들의 약속이 명령어 집합 구조(ISA)입니다. 같은 C 코드라도 어떤 ISA냐에 따라 다른 기계어로 번역됩니다. 설계 철학은 크게 두 가지로 나뉩니다.

  • CISC(예: x86) — 명령이 많고 복잡. 한 명령이 "메모리에서 읽고 더해서 다시 쓰기"까지 한 번에 처리. 명령 수는 적지만 한 명령이 복잡.
  • RISC(예: ARM, RISC-V) — 명령이 적고 단순·균일. 메모리 접근은 load/store 명령으로만. 명령 수는 늘지만 각 명령이 단순해 파이프라이닝·저전력에 유리.

💡 경계는 흐려졌다

오늘날 x86 CPU도 내부에서는 복잡한 명령을 단순한 마이크로 연산(μops)으로 쪼개 RISC처럼 실행합니다. 순수한 구분은 약해졌지만, "단순·균일한 명령이 파이프라이닝과 저전력에 유리하다"는 RISC 철학이 모바일 시대에 ARM의 승리로 이어졌습니다.

엔디안 (Endianness)

여러 바이트로 이루어진 수(예: 4바이트 정수)를 메모리에 저장할 때, 어느 바이트를 먼저 둘 것인가의 약속이 엔디안입니다. 정답은 없고, 시스템마다 다른 관습입니다.

  • 리틀 엔디안 — 낮은 자리(LSB)를 낮은 주소에. x86·ARM 대부분. "거꾸로" 저장되는 것처럼 보임.
  • 빅 엔디안 — 높은 자리(MSB)를 낮은 주소에. 사람이 쓰는 순서와 같음. 네트워크 전송 표준(network byte order).
같은 값도 바이트 순서가 정반대 — 시스템 간 데이터 교환 시 주의
⚠️ 왜 중요한가

엔디안이 다른 두 컴퓨터가 바이트를 그대로 주고받으면 숫자가 깨집니다. 그래서 네트워크 통신은 빅 엔디안으로 통일하고(htonl 등으로 변환), 파일 포맷도 엔디안을 명시합니다. 같은 기계 안에서만 도는 코드라면 신경 쓸 일이 거의 없습니다.

🎯 핵심 정리
  • 컴퓨터는 CPU·메모리·입출력이 버스로 연결된 폰 노이만 구조다.
  • 모든 데이터는 이진수이며, 음수는 2의 보수로 표현해 뺄셈을 덧셈 회로로 처리한다.
  • AND·OR·NOT 같은 논리 게이트의 조합이 CPU의 모든 계산을 이룬다.
  • CPU는 fetch-decode-execute 사이클을 초당 수십억 번 반복한다.
  • 메모리는 속도-용량을 절충한 계층 구조이며, 파이프라이닝으로 명령을 겹쳐 처리해 속도를 높인다.
  • 실수는 IEEE 754(부호·지수·가수)로 표현되며 미세한 오차가 생기므로 직접 등호 비교를 피한다.
  • 캐시는 지역성에 베팅해 캐시 라인 단위로 적재하며, 연속 접근으로 적중률을 높이면 큰 성능 이득을 본다.
  • 가상 메모리는 가상↔물리 주소를 변환해 격리·확장을 제공하고, TLB가 그 변환을 캐싱한다.
  • 인터럽트는 느린 입출력을 기다리지 않게 하고, 병렬성(SIMD·멀티코어)은 동시에 더 많이 처리해 속도를 높인다.
  • ISA는 RISC(단순·균일)와 CISC(복잡·강력)로 나뉘고, 엔디안은 멀티바이트 값의 저장 순서 약속이다.
비트 / 바이트
비트는 0 또는 1 한 자리, 바이트는 8비트 묶음.
2의 보수
비트를 반전하고 1을 더해 음수를 표현하는 방식.
레지스터
CPU 내부의 가장 빠른 초소형 임시 저장 공간.
fetch-decode-execute
명령을 가져오고·해석하고·실행하는 CPU의 기본 반복 사이클.
캐시(Cache)
자주 쓰는 데이터를 CPU 가까이 두는 빠른 임시 메모리.
파이프라이닝
여러 명령의 처리 단계를 겹쳐 실행해 처리량을 높이는 기법.
IEEE 754
실수를 부호·지수·가수로 나눠 저장하는 부동소수점 표준.
지역성(Locality)
방금 쓴 데이터(시간)·그 옆 데이터(공간)를 곧 다시 쓰는 경향. 캐시 효과의 근거.
가상 메모리 / TLB
가상 주소를 물리 주소로 변환하는 체계와, 그 변환을 캐싱하는 CPU 내부 하드웨어.
인터럽트
장치가 작업 완료 등을 CPU에 알려, 하던 일을 잠시 멈추고 처리하게 하는 신호.
SIMD
명령 하나로 여러 데이터를 동시에 처리하는 데이터 병렬 방식.
RISC / CISC
각각 단순·균일한 명령(ARM)과 복잡·강력한 명령(x86)을 지향하는 ISA 설계 철학.
엔디안(Endianness)
멀티바이트 값을 메모리에 저장할 때의 바이트 순서 약속(리틀/빅).
