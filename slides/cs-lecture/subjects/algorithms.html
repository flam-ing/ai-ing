알고리즘

같은 문제도 어떻게 푸느냐에 따라 1초가 걸릴 수도, 한 시간이 걸릴 수도 있습니다. 문제를 똑똑하게 푸는 절차, 알고리즘을 배웁니다.

알고리즘이란?

알고리즘(algorithm)은 문제를 푸는 정확한 절차입니다. "입력을 받아 정해진 단계를 거쳐 원하는 출력을 내는 방법"이라고 생각하면 됩니다. 요리 레시피와 같습니다 — 재료(입력)를 정해진 순서대로 처리해 음식(출력)을 만듭니다.

좋은 알고리즘은 정확하고(올바른 답), 효율적(빠르고 메모리를 적게 씀)이어야 합니다. 같은 문제라도 절차가 영리하면 데이터가 백만 개여도 눈 깜짝할 사이에 풀리고, 그렇지 않으면 한참 걸립니다.

📌 이 페이지에서 배우는 것

시간 복잡도 복습 → 정렬(버블·병합·퀵) → 탐색(선형·이진) → 재귀와 분할정복 → 그리디 → 동적계획법 → 그래프 탐색(BFS·DFS)까지, 알고리즘 사고의 큰 흐름을 잡습니다.

먼저: 시간 복잡도 복습

알고리즘을 비교하려면 "데이터가 많아질 때 얼마나 느려지는가"를 봐야 합니다. 이를 표현하는 공통 언어가 빅오 표기법(Big-O)입니다. 입력 크기 n이 커질 때의 증가 추세만 따집니다.

아래로 갈수록 좋음 — 정렬은 보통 O(n²) vs O(n log n)으로 갈린다
표기이름대표 알고리즘
O(log n)로그이진 탐색
O(n)선형선형 탐색
O(n log n)선형로그병합 정렬, 퀵 정렬(평균)
O(n²)제곱버블 정렬
💡 직관

데이터가 100만 개일 때 O(n²)는 약 1조 번, O(n log n)은 약 2천만 번 연산합니다. 약 5만 배 차이 — 알고리즘 선택이 곧 속도입니다.

정렬 (Sorting)

정렬은 데이터를 크기 순으로 나열하는 일입니다. 가장 기본적이면서 알고리즘 사고를 익히기에 좋은 주제입니다. 세 가지를 비교해 봅니다.

  • 버블 정렬 — 옆 사람끼리 비교해 큰 값을 뒤로 보내기를 반복. 단순하지만 느림(O(n²)).
  • 병합 정렬 — 절반으로 쪼갠 뒤 정렬하며 합치기. 항상 O(n log n).
  • 퀵 정렬 — 기준값(pivot)보다 작은/큰 그룹으로 나눠 재귀. 평균 O(n log n).
버블 정렬: 옆끼리 비교·교환하면 한 패스마다 가장 큰 값이 뒤로 떠오른다
bubble_sort.py
def bubble_sort(a):
    n = len(a)
    for i in range(n - 1):
        for j in range(n - 1 - i):       # 확정된 뒤쪽은 제외
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]   # 교환
    return a

print(bubble_sort([5, 3, 8, 1]))   # [1, 3, 5, 8] — O(n²)
merge_quick.py
def merge_sort(a):
    if len(a) <= 1:
        return a
    mid = len(a) // 2
    left  = merge_sort(a[:mid])       # 절반씩 쪼개고
    right = merge_sort(a[mid:])
    return merge(left, right)         # 정렬하며 합친다 → O(n log n)

def quick_sort(a):
    if len(a) <= 1:
        return a
    pivot = a[len(a) // 2]            # 기준값 하나 선택
    less  = [x for x in a if x < pivot]
    equal = [x for x in a if x == pivot]
    more  = [x for x in a if x > pivot]
    return quick_sort(less) + equal + quick_sort(more)
정렬평균최악특징
버블 정렬O(n²)O(n²)구현이 쉬움, 느림
병합 정렬O(n log n)O(n log n)안정적, 추가 메모리 필요
퀵 정렬O(n log n)O(n²)실무에서 가장 빠른 편

탐색 (Searching)

탐색은 "이 값이 어디 있는가"를 찾는 일입니다. 선형 탐색은 앞에서부터 하나씩 확인하고(O(n)), 이진 탐색은 정렬된 데이터에서 가운데를 보고 절반을 버리기를 반복합니다(O(log n)).

이진 탐색: 가운데를 보고 절반을 버린다 → 7번이면 약 3번 만에 발견
binary_search.py
def binary_search(a, target):       # a는 정렬되어 있어야 함
    lo, hi = 0, len(a) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if a[mid] == target:
            return mid               # 발견
        elif a[mid] < target:
            lo = mid + 1             # 오른쪽 절반만
        else:
            hi = mid - 1             # 왼쪽 절반만
    return -1                        # 없음

print(binary_search([1,3,5,7,9,11,13], 7))   # 3 — O(log n)
⚠️ 전제 조건

이진 탐색은 데이터가 반드시 정렬되어 있어야 작동합니다. 정렬되지 않았다면 선형 탐색을 쓰거나 먼저 정렬해야 합니다.

재귀와 분할정복

재귀(recursion)는 함수가 자기 자신을 다시 호출하는 기법입니다. 큰 문제를 똑같은 모양의 작은 문제로 줄여 풀 때 자연스럽습니다. 단, 더 이상 쪼갤 수 없는 종료 조건(base case)이 반드시 있어야 무한 반복을 피합니다.

분할정복(divide & conquer)은 재귀의 대표 활용법입니다. ① 문제를 절반으로 나누고(divide) ② 각각 풀고(conquer) ③ 합칩니다(combine). 병합 정렬과 퀵 정렬이 바로 이 방식입니다.

recursion.py
def factorial(n):
    if n <= 1:           # 종료 조건 (base case)
        return 1
    return n * factorial(n - 1)   # 자기 자신을 더 작게 호출

print(factorial(5))      # 5*4*3*2*1 = 120
💡 재귀를 읽는 법

"이 함수가 한 단계 작은 문제를 이미 풀었다고 믿어라." factorial(4)가 24를 준다고 믿으면 factorial(5)=5×24는 자명합니다. 종료 조건만 맞으면 나머지는 컴퓨터가 알아서 풀어 줍니다.

그리디 (Greedy)

그리디(탐욕) 알고리즘은 매 순간 가장 좋아 보이는 선택을 하고 절대 되돌아보지 않는 방법입니다. 거스름돈을 줄 때 가장 큰 동전부터 내주는 것이 전형적인 예입니다.

간단하고 빠르지만, "지금의 최선"이 "전체의 최선"과 항상 같지는 않습니다. 그리디가 정답을 보장하는 문제인지 먼저 따져야 합니다.

coin_change.py
def make_change(amount, coins=[500, 100, 50, 10]):
    result = []
    for coin in coins:           # 큰 동전부터
        count = amount // coin   # 최대한 많이 사용
        amount -= coin * count
        result += [coin] * count
    return result

print(make_change(1260))   # [500, 500, 100, 100, 50, 10]
⚠️ 그리디의 함정

동전이 [4, 3, 1]이고 6원을 거슬러야 하면 그리디는 4+1+1(3개)을 택하지만, 정답은 3+3(2개)입니다. 그리디가 틀릴 수 있는 문제는 동적계획법으로 풀어야 합니다.

동적계획법 (DP)

동적계획법(Dynamic Programming)은 겹치는 작은 문제의 답을 한 번만 계산해 저장해 두고 재사용하는 기법입니다. 피보나치를 단순 재귀로 풀면 같은 값을 수없이 다시 계산하지만, DP는 한 번 구한 값을 표에 적어 두고 꺼내 씁니다.

fibonacci_dp.py
# 단순 재귀: 같은 값을 중복 계산 → O(2ⁿ), 매우 느림
def fib_slow(n):
    if n < 2: return n
    return fib_slow(n-1) + fib_slow(n-2)

# DP(메모이제이션): 한 번 구한 값 저장 → O(n)
def fib_fast(n, memo={}):
    if n < 2: return n
    if n in memo: return memo[n]          # 저장돼 있으면 꺼내 씀
    memo[n] = fib_fast(n-1) + fib_fast(n-2)
    return memo[n]

print(fib_fast(50))   # 12586269025 — 즉시 계산
💡 DP가 통하는 조건

① 큰 문제가 작은 문제로 나뉘고(최적 부분 구조), ② 그 작은 문제들이 여러 번 겹쳐 나올 때(중복 부분 문제) DP가 위력을 발휘합니다.

그래프 탐색 (BFS · DFS)

그래프는 점(정점)들을 선(간선)으로 이은 구조입니다. 지하철 노선도, 친구 관계, 미로가 모두 그래프입니다. 그래프를 훑는 두 가지 기본 방법이 있습니다.

  • BFS(너비 우선 탐색) — 가까운 곳부터 물결처럼 퍼져 나감. 큐를 사용. 최단 경로 찾기에 좋음.
  • DFS(깊이 우선 탐색) — 한 방향으로 끝까지 파고든 뒤 되돌아옴. 스택/재귀를 사용.
BFS는 층별로 넓게, DFS는 한 갈래로 깊게 — 방문 순서가 다르다
graph_search.py
from collections import deque

graph = {"A": ["B","C"], "B": ["D","E"], "C": [], "D": [], "E": []}

def bfs(start):
    visited, q = [], deque([start])     # 큐 사용
    while q:
        node = q.popleft()              # 먼저 넣은 것 먼저
        if node not in visited:
            visited.append(node)
            q.extend(graph[node])
    return visited

def dfs(node, visited=None):
    if visited is None: visited = []
    visited.append(node)
    for nxt in graph[node]:             # 재귀로 깊이 우선
        if nxt not in visited:
            dfs(nxt, visited)
    return visited

print(bfs("A"))   # ['A', 'B', 'C', 'D', 'E']
print(dfs("A"))   # ['A', 'B', 'D', 'E', 'C']

힙 정렬 (Heap Sort)

힙(heap)은 "부모가 자식보다 항상 크거나(최대 힙) 작은(최소 힙)" 규칙을 지키는 완전 이진 트리입니다. 이 규칙 덕분에 가장 큰(작은) 값이 늘 맨 위(루트)에 있습니다. 힙 정렬은 이 성질을 이용해, 루트의 값을 하나씩 꺼내며 정렬합니다.

핵심은 힙을 배열로 표현한다는 점입니다. 인덱스 i의 자식은 2i+1, 2i+2이고, 부모는 (i-1)//2입니다. 별도의 노드·포인터 없이 배열만으로 트리를 다룹니다.

최대 힙: 루트가 가장 큰 값. 트리를 배열 [90,70,40,30,50]로 그대로 저장
heap_sort.py
def heapify(a, n, i):           # i를 루트로 하는 부분을 최대 힙으로
    largest = i
    left, right = 2*i + 1, 2*i + 2
    if left < n and a[left] > a[largest]:
        largest = left
    if right < n and a[right] > a[largest]:
        largest = right
    if largest != i:
        a[i], a[largest] = a[largest], a[i]   # 큰 자식과 교환
        heapify(a, n, largest)                # 내려가며 다시 정리

def heap_sort(a):
    n = len(a)
    for i in range(n // 2 - 1, -1, -1):   # ① 전체를 힙으로 만들기
        heapify(a, n, i)
    for end in range(n - 1, 0, -1):       # ② 루트(최댓값)를 뒤로 빼기
        a[0], a[end] = a[end], a[0]
        heapify(a, end, 0)                # 남은 부분만 다시 힙으로
    return a

print(heap_sort([30, 70, 40, 90, 50]))   # [30, 40, 50, 70, 90] — O(n log n)
💡 힙 정렬의 장점

병합 정렬과 달리 추가 배열이 필요 없는 제자리(in-place) 정렬이고, 최악에도 O(n log n)을 보장합니다. 퀵 정렬의 최악(O(n²)) 위험이 없는 안전한 선택입니다. 단, 캐시 효율이 낮아 실제 속도는 퀵 정렬보다 느린 편입니다.

계수 정렬과 기수 정렬

지금까지의 정렬은 원소끼리 비교해서 순서를 정했습니다. 비교 기반 정렬은 이론상 O(n log n)보다 빠를 수 없습니다. 하지만 데이터가 "0~1000 사이 정수" 처럼 제한적이면, 비교 없이 O(n)에 정렬할 수 있습니다.

  • 계수 정렬(Counting Sort) — 각 값이 몇 번 나왔는지 세어 그 개수만큼 순서대로 출력. 값의 범위가 작을 때 강력.
  • 기수 정렬(Radix Sort) — 자릿수별로 계수 정렬을 반복. 일의 자리 → 십의 자리 → … 순으로 정렬.
계수 정렬: 값별 등장 횟수만 세면 비교 없이 정렬된다
counting_sort.py
def counting_sort(a, k):            # a의 값은 0..k 범위라고 가정
    count = [0] * (k + 1)
    for x in a:
        count[x] += 1               # 각 값의 등장 횟수를 센다
    result = []
    for value in range(k + 1):
        result += [value] * count[value]   # 개수만큼 펼친다
    return result

print(counting_sort([2, 0, 2, 1, 3, 0], k=3))   # [0, 0, 1, 2, 2, 3] — O(n+k)
⚠️ 만능은 아니다

계수 정렬은 값의 범위 k가 크면(예: 0~10억) 그만큼 큰 카운트 배열이 필요해 메모리가 폭발합니다. 값의 범위가 원소 개수와 비슷한 수준일 때만 유리합니다. 음수나 실수에는 그대로 쓸 수 없습니다.

다익스트라 최단 경로

BFS는 모든 간선의 "거리"가 1일 때 최단 경로를 줍니다. 하지만 도로처럼 간선마다 가중치(거리·시간·비용)가 다르면 BFS로는 부족합니다. 다익스트라(Dijkstra) 알고리즘은 가중치가 있는 그래프에서 한 출발점부터 모든 곳까지의 최단 거리를 구합니다.

핵심 아이디어는 그리디입니다. "아직 확정 안 된 정점 중 가장 가까운 것"을 골라 확정하고, 그 정점을 거쳐 가면 더 가까워지는 이웃들의 거리를 갱신(완화, relaxation)합니다. 가장 가까운 것부터 확정하므로 한 번 확정된 거리는 다시 줄지 않습니다.

A→D 최단은 A→C(1)→B(3)→D(8). 작은 숫자는 A로부터의 확정 거리
dijkstra.py
import heapq

def dijkstra(graph, start):
    dist = {node: float("inf") for node in graph}
    dist[start] = 0
    pq = [(0, start)]                   # (거리, 정점) 최소 힙
    while pq:
        d, node = heapq.heappop(pq)     # 가장 가까운 정점부터 확정
        if d > dist[node]:
            continue                    # 이미 더 짧은 경로로 처리됨
        for nxt, weight in graph[node]:
            new_d = d + weight
            if new_d < dist[nxt]:       # 더 짧으면 갱신(완화)
                dist[nxt] = new_d
                heapq.heappush(pq, (new_d, nxt))
    return dist

graph = {"A": [("B",4),("C",1)], "B": [("D",5)],
         "C": [("B",2),("D",8)], "D": []}
print(dijkstra(graph, "A"))   # {'A':0, 'B':3, 'C':1, 'D':8}
⚠️ 음수 간선 주의

다익스트라는 간선 가중치가 음수이면 작동하지 않습니다. "가장 가까운 것을 확정하면 더 줄지 않는다"는 전제가 음수 때문에 깨지기 때문입니다. 음수 간선이 있으면 벨만-포드 알고리즘을 씁니다.

최소 신장 트리 (크루스칼 · 프림)

여러 도시를 도로로 모두 잇되 총 공사비를 최소로 하려면 어떤 도로를 골라야 할까요? 모든 정점을 사이클 없이 연결하는 부분 그래프를 신장 트리라 하고, 그 중 간선 가중치 합이 가장 작은 것이 최소 신장 트리(MST)입니다.

  • 크루스칼(Kruskal) — 모든 간선을 가중치 순으로 정렬하고, 사이클을 만들지 않는 간선부터 골라 담는다. (간선 중심)
  • 프림(Prim) — 한 정점에서 시작해, 지금 트리에 닿는 간선 중 가장 싼 것을 계속 붙여 나간다. (정점 중심)
초록 굵은 간선이 MST(합 1+2+3+4=10). 빨강 점선은 사이클을 만들어 제외
kruskal.py
def find(parent, x):                 # 유니온-파인드: 루트 찾기
    while parent[x] != x:
        parent[x] = parent[parent[x]]   # 경로 압축
        x = parent[x]
    return x

def kruskal(n, edges):               # edges = [(가중치, u, v), ...]
    parent = list(range(n))
    mst, total = [], 0
    for w, u, v in sorted(edges):    # 가중치 오름차순
        ru, rv = find(parent, u), find(parent, v)
        if ru != rv:                 # 사이클을 안 만들면 채택
            parent[ru] = rv
            mst.append((u, v, w)); total += w
    return mst, total

edges = [(1,0,1),(2,2,3),(3,0,2),(4,1,4),(5,1,3),(6,3,4)]
print(kruskal(5, edges))   # MST 간선들과 총합 10
💡 어느 것을 쓸까

간선이 적은 희소 그래프면 간선 정렬이 빠른 크루스칼이, 간선이 많은 밀집 그래프면 정점을 키워 가는 프림이 유리합니다. 둘 다 그리디로 같은 최적 MST를 찾습니다.

백트래킹 (Backtracking)

백트래킹은 가능한 모든 경우를 트리처럼 탐색하되, 가망 없는 가지는 일찍 잘라내는(가지치기) 기법입니다. "일단 가 보고, 막히면 되돌아온다(back-track)"가 이름의 뜻입니다. 미로 찾기, 스도쿠, N-퀸이 대표 문제입니다.

N-퀸은 N×N 체스판에 퀸 N개를 서로 공격하지 못하게 놓는 문제입니다. 한 행씩 내려가며 퀸을 놓아 보고, 충돌하면 그 가지를 버리고 다른 열을 시도합니다.

백트래킹: 선택 → 막히면 취소(back) → 다음 선택. 가망 없는 가지는 자른다
n_queens.py
def solve_n_queens(n):
    cols = [-1] * n          # cols[r] = r행에 놓은 퀸의 열
    count = 0

    def safe(r, c):
        for prev in range(r):           # 이미 놓은 윗줄들과 비교
            pc = cols[prev]
            if pc == c or abs(pc - c) == abs(prev - r):  # 같은 열·대각선
                return False
        return True

    def place(r):
        nonlocal count
        if r == n:              # 모든 행에 성공적으로 놓음
            count += 1
            return
        for c in range(n):      # 현재 행의 각 열을 시도
            if safe(r, c):
                cols[r] = c     # 선택
                place(r + 1)    # 다음 행으로
                cols[r] = -1    # 취소(back-track)

    place(0)
    return count

print(solve_n_queens(8))   # 92 — 8-퀸의 모든 배치 수
💡 가지치기가 핵심

모든 경우를 무작정 다 만들면 N=8에서 8⁸≈1700만 가지입니다. safe()로 충돌을 미리 걸러 가지를 자르면 실제 탐색은 그 일부에 불과합니다. 백트래킹의 성능은 "얼마나 일찍 가망 없는 가지를 버리는가"에 달려 있습니다.

비트마스킹 기법

비트마스킹은 정수 하나의 각 비트를 켜짐/꺼짐 플래그로 사용해 "집합"을 표현하는 기법입니다. 예를 들어 5명의 참석 여부를 {0,1,1,0,1} 대신 정수 0b01101 하나로 다룹니다. 집합 연산이 비트 연산 한 번으로 끝나 매우 빠르고 메모리를 거의 안 씁니다.

연산코드의미
원소 추가s | (1 << i)i번째 비트를 켠다
원소 제거s & ~(1 << i)i번째 비트를 끈다
포함 확인s & (1 << i)i번째 비트가 켜졌나
토글s ^ (1 << i)i번째 비트를 뒤집는다
원소 개수bin(s).count("1")켜진 비트 수
bitmask.py
s = 0                     # 빈 집합 (00000)
s |= (1 << 0)             # 0번 추가  → 00001
s |= (1 << 2)             # 2번 추가  → 00101

print(s & (1 << 2))       # 0이 아니면 포함 → True 취급
print(bin(s).count("1"))  # 원소 개수 → 2

# 부분집합 전체 순회: 원소 3개짜리 집합의 모든 부분집합
full = 0b111
sub = full
while sub > 0:
    print(format(sub, "03b"))  # 111, 110, 101, 100, 011, 010, 001
    sub = (sub - 1) & full
💡 어디에 쓰나

"방문한 도시 집합" 같은 상태를 정수로 압축해 DP 키로 쓰는 비트마스킹 DP(예: 외판원 문제 TSP)에 핵심입니다. 원소가 약 20개 이하일 때, 2²⁰개의 상태를 배열 인덱스로 곧장 다룰 수 있습니다.

분할상환 분석 (Amortized Analysis)

어떤 연산은 대부분 빠르지만 가끔 한 번 크게 느린 경우가 있습니다. 매번 최악(O(n))으로 평가하면 너무 비관적입니다. 분할상환 분석은 "연속된 여러 연산의 총비용을 횟수로 나눈, 1회당 평균 비용"을 따져 더 현실적으로 평가합니다.

대표 예가 파이썬 list.append입니다. 배열이 꽉 차면 더 큰 배열을 새로 만들어 전부 복사(O(n))하지만, 보통 크기를 2배로 늘리므로 이 비싼 복사는 드물게 일어납니다. 그래서 append 한 번의 분할상환 비용은 O(1)입니다.

대부분 싸고(초록) 가끔 비싼(빨강) 확장. 평균을 내면 O(1)
💡 평균 복잡도와의 차이

평균 복잡도는 "입력이 무작위일 때의 기댓값"이고, 분할상환은 "최악 입력이라도 연산을 길게 이어 보면 1회당 평균 비용"입니다. 분할상환은 운에 기대지 않는, 더 강한 보장입니다.

🎯 핵심 정리
  • 알고리즘은 빅오로 비교한다 — 정렬·탐색에서 O(n²)와 O(n log n)의 차이가 결정적이다.
  • 정렬은 병합·퀵이 O(n log n)으로 빠르고, 이진 탐색은 정렬된 데이터에서 O(log n)이다.
  • 재귀는 종료 조건이 핵심이며, 분할정복은 나누고-풀고-합치는 재귀 패턴이다.
  • 그리디는 빠르지만 항상 정답은 아니다 — 정답 보장 여부를 먼저 따진다.
  • DP는 겹치는 부분 문제의 답을 저장해 재사용하고, 그래프는 BFS(넓게)·DFS(깊게)로 훑는다.
  • 힙 정렬은 제자리·최악 O(n log n)을 보장하고, 계수·기수 정렬은 값 범위가 작으면 O(n)에 정렬한다.
  • 가중치 그래프의 최단 경로는 다익스트라(음수 간선 불가), 최소 연결 비용은 MST(크루스칼·프림)로 푼다.
  • 백트래킹은 모든 경우를 탐색하되 가망 없는 가지를 일찍 잘라(가지치기) 효율을 얻는다.
  • 비트마스킹은 집합을 정수로 압축해 빠르게 다루고, 분할상환 분석은 가끔 비싼 연산의 1회당 평균 비용을 본다.
빅오(Big-O)
입력이 커질 때 연산량이 증가하는 추세를 나타내는 표기법.
피벗(Pivot)
퀵 정렬에서 작은/큰 그룹을 나누는 기준이 되는 값.
종료 조건(Base case)
재귀가 더 이상 자신을 호출하지 않고 멈추는 가장 작은 경우.
메모이제이션
한 번 계산한 결과를 저장해 두고 다시 필요할 때 재사용하는 DP 기법.
BFS / DFS
각각 너비 우선(큐)·깊이 우선(스택·재귀) 그래프 탐색 방법.
힙(Heap)
루트에 항상 최대(또는 최소)값이 오는 완전 이진 트리. 배열로 표현하며 우선순위 큐의 기반.
완화(Relaxation)
최단 경로에서 "더 짧은 경로를 찾으면 거리를 갱신"하는 핵심 연산.
최소 신장 트리(MST)
모든 정점을 사이클 없이 잇되 간선 가중치 합이 최소인 부분 그래프.
백트래킹
모든 경우를 트리로 탐색하되 가망 없는 가지를 잘라 되돌아가는(back-track) 기법.
비트마스킹
정수의 각 비트를 플래그로 써서 집합을 표현·연산하는 기법.
분할상환 분석
연속된 연산의 총비용을 횟수로 나눈 1회당 평균 비용을 따지는 분석법.
