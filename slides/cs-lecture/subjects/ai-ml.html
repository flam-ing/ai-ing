인공지능 / 머신러닝

규칙을 일일이 적는 대신, 데이터로부터 규칙을 스스로 찾게 하는 기술입니다. 컴퓨터가 "경험을 통해 배우는" 원리를 차근차근 풀어봅니다.

AI와 머신러닝이란?

인공지능(AI, Artificial Intelligence)은 사람의 지능적인 행동(인식·판단·학습)을 컴퓨터로 흉내 내려는 넓은 분야입니다. 머신러닝(ML, Machine Learning)은 그중에서도 "규칙을 직접 코딩하지 않고, 데이터를 보고 규칙을 스스로 학습하는" 방법입니다.

요리에 비유하면, 전통적 프로그래밍은 레시피(규칙)를 사람이 다 적어 주는 것이고, 머신러닝은 완성된 요리(데이터) 수천 접시를 보여 주며 "레시피를 네가 알아내 봐"라고 시키는 것입니다.

방향이 거꾸로: 규칙을 입력하는 게 아니라 규칙을 출력으로 얻는다

학습의 세 가지 방식

머신러닝은 "무엇을 보고 배우느냐"에 따라 크게 세 갈래로 나뉩니다.

  • 지도 학습(Supervised) — 입력과 정답(라벨)이 함께 주어집니다. "이 사진은 고양이"처럼 답을 알려 주며 가르칩니다. 예: 스팸 분류, 집값 예측.
  • 비지도 학습(Unsupervised) — 정답 없이 데이터만 줍니다. 비슷한 것끼리 묶기(군집화)나 숨은 구조 찾기. 예: 고객 그룹 분류.
  • 강화 학습(Reinforcement) — 행동에 대한 보상/벌점으로 배웁니다. 시행착오를 통해 점수를 높이는 전략을 익힘. 예: 게임 AI, 로봇 제어.
방식주어지는 것목표예시
지도 학습입력 + 정답입력→정답 예측스팸 필터, 가격 예측
비지도 학습입력만구조·그룹 발견고객 군집화
강화 학습보상 신호누적 보상 최대화바둑·게임 AI
💡 한 줄 구분법

"정답표가 있으면 지도, 없으면 비지도, 점수를 보고 행동을 고치면 강화"로 기억하세요.

선형회귀 (Linear Regression)

가장 단순한 지도 학습입니다. 점들을 가장 잘 지나는 직선 하나를 찾는 일이에요. 예를 들어 "공부 시간(x)에 따른 시험 점수(y)"를 모았다면, 그 관계를 y = w·x + b 라는 직선으로 표현합니다. 여기서 w(기울기)와 b(절편)가 학습으로 찾아야 할 값입니다.

점(실제 데이터)과 직선(예측) 사이의 오차 합을 가장 작게 만드는 직선을 찾는다

"가장 잘 맞는" 기준은 오차(예측값과 실제값의 차이)입니다. 오차를 제곱해 모두 더한 값을 손실(loss)이라 하고, 이 손실이 최소가 되도록 w와 b를 조금씩 조정합니다. 이 조정 과정이 경사 하강법(gradient descent)입니다.

linear_regression.py
# 개념용 의사 코드 — 경사 하강법으로 직선 학습
w, b = 0.0, 0.0           # 시작값 (아무거나)
lr = 0.01                 # 학습률: 한 번에 얼마나 조정할지

for step in range(1000):
    y_pred = w * x + b              # 현재 직선의 예측
    error  = y_pred - y_true        # 오차
    # 오차를 줄이는 방향으로 w, b 를 조금씩 이동
    w -= lr * mean(error * x)
    b -= lr * mean(error)

print(w, b)               # 학습된 기울기와 절편

분류와 결정경계 (Classification)

선형회귀가 "숫자"를 예측한다면, 분류는 "어느 그룹인가"를 예측합니다. 예: 메일이 스팸/정상인지, 사진이 고양이/강아지인지. 모델은 데이터 공간에 결정경계(decision boundary)라는 선을 그어, 그 선을 기준으로 그룹을 나눕니다.

선 한쪽은 A 그룹, 다른 쪽은 B 그룹 — 새 점이 들어오면 어느 쪽인지로 판정

분류 결과를 "확률"로 부드럽게 출력하기 위해 시그모이드(sigmoid) 같은 함수를 씁니다. 값이 0~1 사이로 눌려, "스팸일 확률 0.92"처럼 해석할 수 있습니다.

classify.py
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))     # 0~1 사이 확률로 변환

def predict(x, w, b):
    z = np.dot(w, x) + b
    prob = sigmoid(z)               # 예: 0.92
    return "스팸" if prob >= 0.5 else "정상"

신경망 기초: 퍼셉트론

퍼셉트론(perceptron)은 신경망의 가장 작은 단위로, 뇌의 뉴런 하나를 흉내 낸 것입니다. 여러 입력에 각각 가중치(weight)를 곱해 더하고, 그 합이 기준을 넘으면 신호를 내보냅니다(활성화).

입력 × 가중치의 합 → 활성화 함수 → 출력. 학습은 가중치를 조정하는 일

퍼셉트론 하나는 직선 하나만 그을 수 있어 단순한 문제만 풉니다. 하지만 이것을 여러 층으로 쌓으면(다층 신경망) 휘어진 복잡한 경계도 표현할 수 있습니다.

perceptron.py
import numpy as np

def perceptron(x, w, b):
    z = np.dot(w, x) + b      # 입력 × 가중치 합 + 편향
    return 1 if z > 0 else 0  # 계단 함수: 넘으면 1, 아니면 0

# AND 게이트 예시: 둘 다 1일 때만 1
w = np.array([0.5, 0.5])
b = -0.7
print(perceptron([1, 1], w, b))  # 1
print(perceptron([1, 0], w, b))  # 0

과적합과 학습

모델이 학습 데이터를 너무 달달 외워, 새로운 데이터에서는 오히려 틀리는 현상을 과적합(overfitting)이라 합니다. 시험 족보만 통째로 외운 학생이 처음 보는 응용 문제에서 막히는 것과 같아요.

학습 오차는 계속 줄지만, 어느 순간부터 새 데이터 오차는 다시 늘어난다

방지법: 데이터를 학습용/검증용으로 나눠 점검하기, 모델을 너무 복잡하게 만들지 않기(규제, regularization), 데이터를 더 모으기, 학습을 적당한 시점에 멈추기(early stopping).

⚠️ 과소적합(underfitting)도 있다

반대로 모델이 너무 단순해 학습 데이터조차 못 맞히면 과소적합입니다. 둘 사이의 균형을 찾는 게 학습의 핵심입니다.

딥러닝 개요

딥러닝(deep learning)은 퍼셉트론(뉴런)을 여러 층으로 깊게 쌓은 신경망입니다. 층이 깊어질수록, 앞 층은 단순한 특징(선·모서리)을, 뒤 층은 복잡한 특징(눈·얼굴)을 잡아내며 점점 추상적인 패턴을 학습합니다.

  • CNN(합성곱 신경망) — 이미지에 강함. 사진 분류, 의료 영상.
  • RNN / 트랜스포머 — 순서가 있는 데이터(글·음성)에 강함. 번역, 챗봇.

딥러닝이 잘 작동하려면 보통 많은 데이터와 강한 연산 능력(GPU)이 필요합니다. 이것이 최근에야 딥러닝이 폭발적으로 발전한 이유입니다.

tiny_net.py
# 개념용: 2개 층을 가진 작은 신경망의 순전파
import numpy as np

def relu(z): return np.maximum(0, z)   # 음수는 0으로 (활성화 함수)

def forward(x, W1, b1, W2, b2):
    h = relu(W1 @ x + b1)   # 은닉층: 특징 추출
    y = W2 @ h + b2         # 출력층: 최종 예측
    return y
📌 큰 그림

딥러닝도 결국 "예측 → 오차 측정 → 가중치 조정"의 반복입니다. 규모와 층이 커졌을 뿐, 기본 원리는 앞에서 본 선형회귀·퍼셉트론과 같습니다.

경사 하강법 자세히 보기

앞에서 학습은 "손실을 줄이는 방향으로 가중치를 조정하는 일"이라고 했습니다. 그 방향을 어떻게 아는 걸까요? 경사 하강법(gradient descent)은 산에서 안개 속에 갇힌 등산객이 발밑의 기울기만 느끼며 가장 가파른 내리막으로 한 걸음씩 내려가는 것과 같습니다. 손실 함수라는 산에서 가장 낮은 골짜기(손실 최소)를 찾아가는 것이죠.

기울기(경사)의 반대 방향으로 조금씩 이동 → 손실의 골짜기에 도달

한 걸음의 크기를 정하는 값이 학습률(learning rate)입니다. 너무 크면 골짜기를 건너뛰어 발산하고, 너무 작으면 한없이 느립니다. 적당한 학습률을 고르는 게 중요합니다.

학습률증상비유
너무 큼손실이 출렁이거나 발산보폭이 커서 골짜기를 펄쩍 건너뜀
적당함안정적으로 손실 감소꾸준히 내리막을 내려감
너무 작음학습이 매우 느림종종걸음이라 도착이 한참

데이터가 많을 때는 전부 보고 한 걸음 내딛는 대신(배치 경사 하강), 일부만 보고 자주 걷는 미니배치/확률적 경사 하강법(SGD)을 씁니다. 더 자주, 더 빠르게 골짜기로 향합니다.

gradient_descent.py
# 개념용: 손실 f(w) = (w - 3)^2 의 최소(w=3) 찾기
def grad(w):
    return 2 * (w - 3)        # 기울기(미분값)

w = 0.0                       # 시작점
lr = 0.1                      # 학습률
for step in range(30):
    w -= lr * grad(w)         # 기울기 반대 방향으로 한 걸음
print(round(w, 3))           # 3.0 에 수렴
💡 기울기 = 방향 안내

기울기는 "여기서 손실이 가장 빨리 커지는 방향"입니다. 그 반대로 가야 손실이 줄어들기 때문에 코드에서 -= 로 빼는 것입니다.

로지스틱 회귀 (분류의 기본기)

이름에 "회귀"가 붙어 헷갈리지만, 로지스틱 회귀(logistic regression)는 사실 가장 널리 쓰이는 분류 알고리즘입니다. 선형회귀처럼 z = w·x + b 를 계산한 뒤, 그 값을 시그모이드에 통과시켜 0~1 사이의 확률로 바꿉니다. 그 확률이 0.5를 넘으면 한 그룹, 아니면 다른 그룹으로 판정합니다.

선형회귀는 "오차 제곱"을 손실로 쓰지만, 분류에서는 확률을 다루기에 더 알맞은 교차 엔트로피(cross-entropy) 손실을 씁니다. "정답을 높은 확률로 맞히면 손실이 작고, 자신 있게 틀리면 손실이 크게" 벌점을 줍니다.

logistic.py
import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

def predict_proba(X, w, b):
    z = X @ w + b
    return sigmoid(z)            # 0~1 확률

def cross_entropy(y_true, p):
    eps = 1e-9                   # log(0) 방지
    return -np.mean(y_true*np.log(p+eps) + (1-y_true)*np.log(1-p+eps))

# 확률 → 분류
p = predict_proba(X, w, b)
labels = (p >= 0.5).astype(int)   # 0 또는 1
📌 선형회귀 vs 로지스틱 회귀

선형회귀는 숫자(집값·점수)를 예측하고, 로지스틱 회귀는 확률·분류(합격/불합격)를 예측합니다. 둘 다 w·x+b에서 출발하지만 마지막 처리와 손실이 다릅니다.

의사결정 트리와 랜덤 포레스트

의사결정 트리(decision tree)는 스무고개처럼 질문을 이어가며 답을 좁히는 모델입니다. "나이가 30 이상인가?", "소득이 높은가?" 같은 질문(분기)을 거치며 잎(leaf)에 도달하면 거기에 예측 결과가 있습니다. 사람이 읽고 이해하기 쉬운 게 큰 장점입니다.

질문을 따라 내려가다 잎에 도달하면 그것이 예측 결과

하지만 트리 하나는 데이터를 너무 잘 외워(과적합) 새 데이터에 약합니다. 그래서 랜덤 포레스트(random forest)는 서로 조금씩 다른 여러 그루의 트리를 만들어 다수결(분류)이나 평균(회귀)으로 결정합니다. "여러 전문가의 의견을 모으면 한 명보다 낫다"는 앙상블(ensemble) 아이디어입니다.

forest.py
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=100)  # 트리 100그루
model.fit(X_train, y_train)        # 학습
pred = model.predict(X_test)       # 다수결로 예측

# 어떤 특징이 중요했는지도 알 수 있다
print(model.feature_importances_)
💡 표 형태 데이터의 강자

엑셀 표 같은 정형 데이터에서는 랜덤 포레스트나 그래디언트 부스팅(XGBoost 등)이 딥러닝보다 잘 통하는 경우가 많습니다. 무조건 신경망이 정답은 아닙니다.

KNN과 SVM 개념

분류 알고리즘에는 직관적인 두 가지가 더 있습니다.

  • KNN(K-최근접 이웃) — "끼리끼리"의 원리입니다. 새 데이터가 들어오면 가장 가까운 이웃 K개를 찾아 그들의 다수결로 분류합니다. 학습이랄 게 거의 없고(데이터를 그냥 기억), 예측할 때 거리를 계산합니다. 단순하지만 데이터가 많으면 느립니다.
  • SVM(서포트 벡터 머신) — 두 그룹을 가르는 가장 넓은 여백(margin)을 가진 경계선을 찾습니다. 경계에 가장 가까운 점들(서포트 벡터)이 선의 위치를 결정합니다. 여백을 넓게 잡을수록 새 데이터에도 잘 일반화됩니다.
KNN은 이웃 투표, SVM은 여백이 가장 넓은 경계를 찾는다
📌 "거리"의 중요성

KNN·SVM 모두 거리를 다루므로, 특징(feature)들의 크기(스케일)를 맞춰주는 정규화가 중요합니다. 키(cm)와 몸무게(kg)처럼 단위가 다르면 큰 숫자가 거리를 지배해 버립니다.

교차 검증과 하이퍼파라미터

모델이 진짜 잘하는지 확인하려면, 학습에 쓰지 않은 데이터로 시험을 봐야 합니다. 보통 데이터를 학습용/검증용/시험용으로 나누지만, 데이터가 적으면 한 번의 분할이 운에 좌우됩니다. 그래서 교차 검증(cross-validation)을 씁니다.

K-폴드 교차 검증은 데이터를 K조각으로 나눠, 한 조각씩 돌아가며 시험용으로 쓰고 나머지로 학습합니다. K번 시험 본 점수의 평균을 성능으로 봅니다. 모든 데이터가 한 번씩 시험에 쓰여 더 믿을 만합니다.

매번 다른 조각을 검증에 사용 → 5개 점수의 평균이 곧 성능

하이퍼파라미터(hyperparameter)는 학습률·트리 개수·이웃 수 K처럼 사람이 미리 정해 주는 설정값입니다(가중치처럼 학습으로 얻는 값과 구분). 여러 후보를 시험해 가장 좋은 조합을 찾는 과정을 그리드 서치/랜덤 서치라 합니다.

cv_search.py
from sklearn.model_selection import cross_val_score, GridSearchCV
from sklearn.ensemble import RandomForestClassifier

# 5-폴드 교차 검증 점수
scores = cross_val_score(RandomForestClassifier(), X, y, cv=5)
print(scores.mean())     # 평균 정확도

# 하이퍼파라미터 후보 탐색
grid = {"n_estimators": [50, 100, 200], "max_depth": [4, 8, None]}
search = GridSearchCV(RandomForestClassifier(), grid, cv=5)
search.fit(X, y)
print(search.best_params_)   # 가장 좋았던 조합
⚠️ 시험지를 미리 보지 말 것

최종 시험용 데이터(test set)는 마지막 한 번만 봐야 합니다. 그것을 보며 하이퍼파라미터를 고치면, 사실상 시험지를 외운 셈이라 성능이 부풀려집니다.

평가지표와 혼동행렬

"정확도(accuracy)"만으로는 부족할 때가 많습니다. 예를 들어 환자 1%만 양성인 병을 모두 "음성"이라고 찍어도 정확도는 99%지만, 정작 환자를 한 명도 못 찾습니다. 그래서 혼동행렬(confusion matrix)로 결과를 네 칸으로 쪼개 봅니다.

TP·TN은 맞힌 것, FP(거짓 경보)·FN(놓침)은 두 종류의 실수
  • 정밀도(Precision) = TP / (TP+FP) — "양성이라 한 것 중 진짜 양성 비율". 거짓 경보를 줄이고 싶을 때 중요(예: 스팸으로 분류한 게 진짜 스팸인가).
  • 재현율(Recall) = TP / (TP+FN) — "실제 양성 중 찾아낸 비율". 놓치면 안 될 때 중요(예: 암 환자를 빠뜨리지 않기).
  • F1 점수 = 정밀도와 재현율의 조화평균 — 둘의 균형을 한 숫자로.
metrics.py
from sklearn.metrics import confusion_matrix, precision_score, recall_score, f1_score

print(confusion_matrix(y_true, y_pred))   # [[TN, FP], [FN, TP]]
print(precision_score(y_true, y_pred))    # 정밀도
print(recall_score(y_true, y_pred))       # 재현율
print(f1_score(y_true, y_pred))           # F1
💡 정밀도와 재현율은 줄다리기

판정 기준(임계값)을 높이면 정밀도가 오르지만 재현율은 떨어지고, 낮추면 반대가 됩니다. 무엇이 더 치명적인지(놓침 vs 거짓 경보)에 따라 균형점을 정합니다.

CNN과 RNN 개요

딥러닝은 데이터의 종류에 따라 알맞은 구조가 있습니다. CNN은 이미지에, RNN은 순서 있는 데이터에 강합니다.

CNN (합성곱 신경망)

이미지를 한 픽셀씩 따로 보면 의미가 없습니다. CNN은 작은 필터(커널)를 이미지 위로 미끄러뜨리며 국소적인 특징(모서리·곡선·무늬)을 잡아냅니다. 층이 깊어질수록 단순한 선 → 눈·코 → 얼굴처럼 점점 복잡한 특징을 조합합니다.

필터로 특징 추출 → 풀링으로 요약 → 최종 분류. 사진 인식의 표준 구조

RNN (순환 신경망)

문장·음성·주가처럼 순서가 중요한 데이터는 앞 내용을 기억해야 합니다. RNN은 이전 단계의 출력을 다음 단계로 되먹임해 "기억"을 흘려보냅니다. 다만 긴 문장에서는 앞부분을 잊는 약점이 있어, 이를 개선한 LSTM/GRU가 나왔고, 오늘날에는 대부분 트랜스포머로 대체되고 있습니다.

📌 데이터에 맞는 구조

이미지=CNN, 순서 데이터=RNN/트랜스포머가 출발점입니다. 핵심 원리(예측→오차→가중치 조정)는 모두 같고, "데이터의 모양에 맞춰 연결을 설계"한 것이 차이입니다.

트랜스포머와 LLM 입문

요즘 화제인 ChatGPT, Claude 같은 대규모 언어 모델(LLM, Large Language Model)은 모두 트랜스포머(Transformer)라는 구조에 기반합니다. 핵심 아이디어는 어텐션(attention)입니다. 문장의 모든 단어를 한꺼번에 보면서, 각 단어가 어떤 다른 단어에 주목해야 하는지를 학습합니다.

예를 들어 "그 동물은 길을 건너지 못했다. 너무 피곤했기 때문이다"에서, "피곤"이 "동물"을 가리킨다는 걸 어텐션이 연결합니다. RNN처럼 순서대로 처리하지 않고 한 번에 보기 때문에, 길고 멀리 떨어진 관계도 잘 잡고 병렬 처리로 학습이 빠릅니다.

  • 토큰화 — 문장을 단어/조각(토큰) 단위로 쪼갭니다.
  • 임베딩 — 각 토큰을 의미를 담은 숫자 벡터로 바꿉니다.
  • 어텐션 — 토큰들이 서로 얼마나 관련 있는지 가중치를 계산합니다.
  • 다음 토큰 예측 — "지금까지의 글 다음에 올 단어"를 확률로 예측하며, 이를 반복해 문장을 생성합니다.
next_token.py
# 개념용: LLM은 "다음 토큰의 확률"을 반복 예측해 글을 만든다
prompt = "오늘 날씨가 정말"
for _ in range(5):
    probs = model.next_token_probs(prompt)   # 각 후보 토큰의 확률
    next_tok = pick(probs)                    # 확률에 따라 하나 선택
    prompt += next_tok                        # 이어붙이고 다시 예측
print(prompt)   # "오늘 날씨가 정말 좋아서 산책을 ..."
⚠️ LLM은 "사실 기계"가 아니다

LLM은 다음에 올 그럴듯한 단어를 예측하는 모델이라, 틀린 내용을 자신 있게 말할 수 있습니다(환각, hallucination). 중요한 사실은 반드시 따로 확인하세요.

🎯 핵심 정리
  • 머신러닝은 규칙을 코딩하는 대신 데이터에서 규칙(모델)을 학습한다.
  • 지도(정답 있음)·비지도(정답 없음)·강화(보상으로 학습)로 나뉜다.
  • 학습은 "예측 → 오차 측정 → 가중치 조정"의 반복(경사 하강법)이다.
  • 분류는 결정경계를 그어 그룹을 나누고, 확률로 결과를 해석한다.
  • 모델을 너무 외우게 하면 과적합 — 검증·규제로 균형을 잡는다.
  • 경사 하강법은 기울기 반대로 한 걸음씩 내려가며, 학습률이 보폭을 정한다.
  • 로지스틱 회귀는 시그모이드로 확률을 내는 기본 분류기다.
  • 의사결정 트리는 해석이 쉽고, 랜덤 포레스트는 여러 트리를 모아 더 강하다.
  • KNN은 이웃 투표, SVM은 여백이 최대인 경계로 분류한다.
  • 교차 검증으로 성능을 믿을 만하게 재고, 하이퍼파라미터는 사람이 튜닝한다.
  • 정확도만 보지 말고 정밀도·재현율·F1·혼동행렬로 모델을 평가한다.
  • CNN은 이미지, RNN/트랜스포머는 순서 데이터에 강하다.
  • LLM은 트랜스포머의 어텐션으로 다음 토큰을 예측해 글을 생성한다.
가중치(Weight)
각 입력이 결과에 미치는 영향의 크기. 학습으로 조정되는 핵심 값.
손실(Loss)
예측이 정답과 얼마나 다른지를 숫자로 나타낸 값. 작을수록 좋다.
경사 하강법
손실을 줄이는 방향으로 가중치를 조금씩 옮겨가며 학습하는 방법.
활성화 함수
뉴런의 출력을 비선형으로 바꿔, 복잡한 패턴을 표현하게 하는 함수.
과적합(Overfitting)
학습 데이터에만 지나치게 맞춰져 새 데이터에서 성능이 떨어지는 현상.
학습률(Learning rate)
경사 하강법에서 한 번에 가중치를 얼마나 옮길지 정하는 보폭 값.
로지스틱 회귀
시그모이드로 확률을 내어 분류하는 기본 알고리즘. 이름과 달리 분류용.
앙상블(Ensemble)
여러 모델의 예측을 모아(다수결·평균) 더 좋은 결과를 내는 기법. 랜덤 포레스트가 대표.
하이퍼파라미터
학습으로 얻는 가중치와 달리, 사람이 미리 정하는 설정값(학습률·트리 수 등).
교차 검증
데이터를 여러 조각으로 나눠 번갈아 검증해 성능을 믿을 만하게 재는 방법.
정밀도 / 재현율
각각 "양성이라 한 것 중 맞은 비율", "실제 양성 중 찾은 비율".
혼동행렬
TP·FP·FN·TN 네 칸으로 분류 결과를 쪼개 보는 표.
CNN
필터로 이미지의 국소 특징을 잡아내는 합성곱 신경망.
어텐션 / 트랜스포머
단어들이 서로 얼마나 주목할지 계산하는 방식과 그 구조. LLM의 핵심.
LLM
다음 토큰을 예측해 글을 생성하는 대규모 언어 모델.
