데이터 과학 / 빅데이터

데이터에서 쓸모 있는 사실을 끄집어내는 일입니다. 모으고, 닦고, 들여다보고, 그림으로 설명하는 전 과정을 입문자 눈높이로 따라갑니다.

데이터 과학이란?

데이터 과학(data science)은 데이터에서 의미 있는 패턴과 결정을 끌어내는 학문입니다. 통계학, 프로그래밍, 그리고 문제가 되는 분야(도메인) 지식이 만나는 지점입니다.

요리에 비유하면, 데이터는 시장에서 사 온 재료입니다. 흙 묻은 채소를 씻고(정제), 맛을 보고(탐색), 손님에게 보기 좋게 담아내는(시각화) 전 과정이 데이터 과학입니다. 분석 결과를 자동으로 학습하게 만들면 그것이 머신러닝으로 이어집니다.

데이터의 종류와 수집

데이터는 모양에 따라 다루는 방법이 다릅니다. 크게 정형(structured) 데이터는 표처럼 행과 열로 깔끔하게 정리된 것(엑셀, 데이터베이스), 비정형(unstructured) 데이터는 텍스트·이미지·음성처럼 표로 담기 어려운 것입니다. 그 중간을 반정형(JSON, 로그)이라 합니다.

또한 값의 성격에 따라 수치형(키, 가격), 범주형(성별, 등급)으로 나눕니다. 수집은 파일·데이터베이스·웹 API·크롤링·센서 등 다양한 출처에서 이뤄집니다.

구분예시저장 형태
정형판매 기록, 회원 명부표(CSV·DB 테이블)
반정형API 응답, 서버 로그JSON·XML
비정형리뷰 텍스트, 사진파일·blob

df = pd.read_csv("sales.csv")

print(df.shape)        # (행 개수, 열 개수)
print(df.dtypes)       # 각 열의 자료형(수치형/범주형) 확인
print(df.head())       # 위에서 5행만 미리 보기

데이터 정제 (결측치·이상치)

현실의 데이터는 거의 항상 지저분합니다. 빈칸(결측치), 말도 안 되는 값(이상치), 중복된 행이 섞여 있죠. "쓰레기를 넣으면 쓰레기가 나온다"(garbage in, garbage out)는 말처럼, 정제는 분석의 신뢰도를 좌우하는 가장 중요한 단계입니다.

  • 결측치(missing value) — 값이 비어 있는 칸. 행을 지우거나, 평균·중앙값으로 채우는 대치(imputation)를 합니다.
  • 이상치(outlier) — 나이가 250살처럼 정상 범위를 크게 벗어난 값. 입력 오류일 수도, 진짜 특이 사례일 수도 있습니다.
clean.py
# 1) 결측치 확인 — 각 열에 빈칸이 몇 개인지
print(df.isnull().sum())

# 2) 나이 열의 결측치를 중앙값으로 채우기(대치)
df["age"] = df["age"].fillna(df["age"].median())

# 3) 중복 행 제거
df = df.drop_duplicates()

# 4) 이상치 걸러내기 — 나이가 0~120 범위 밖이면 제외
df = df[(df["age"] >= 0) & (df["age"] <= 120)]
⚠️ 평균으로 채우기의 함정

결측치를 무조건 평균으로 채우면 분포가 인위적으로 좁아져 분석이 왜곡될 수 있습니다. 왜 비어 있는지를 먼저 생각하고, 중앙값·예측값 등 상황에 맞는 방법을 고르세요.

탐색적 데이터 분석 (EDA)

EDA(Exploratory Data Analysis)는 본격 분석 전에 데이터와 친해지는 단계입니다. 통계 요약과 간단한 그래프로 "이 데이터가 어떻게 생겼는지" 감을 잡습니다. 1977년 통계학자 존 튜키(John Tukey)가 강조한 접근으로, 가설을 세우기 전에 데이터가 말하게 두는 것이 핵심입니다.

eda.py
# 수치형 열들의 요약 통계 한 번에 보기
print(df.describe())   # 개수·평균·표준편차·최소/최대·사분위수

# 범주형 열의 값 분포 세기
print(df["grade"].value_counts())

# 두 열의 관계 한눈에 — 그룹별 평균
print(df.groupby("grade")["score"].mean())
💡 EDA의 자세

"이 데이터로 뭘 증명하자"가 아니라 "이 데이터는 무엇을 말하고 있나"로 시작하세요. 의외의 패턴이나 잘못된 값은 대부분 EDA에서 처음 발견됩니다.

기술통계 (평균·분산·분포)

기술통계(descriptive statistics)는 많은 숫자를 몇 개의 대표값으로 요약하는 도구입니다.

  • 평균(mean) — 모든 값을 더해 개수로 나눈 값. 이상치에 약합니다.
  • 중앙값(median) — 크기 순으로 줄 세웠을 때 한가운데 값. 이상치에 강합니다.
  • 분산(variance)·표준편차(std) — 값들이 평균에서 얼마나 흩어져 있는지.
종 모양으로 가운데가 높은 분포(정규분포). 막대 = 구간별 데이터 개수
📌 평균 vs 중앙값

연봉처럼 한쪽으로 치우친 데이터는 평균이 소수의 고액 연봉에 끌려 올라갑니다. 이럴 땐 중앙값이 "보통 사람"을 더 잘 대표합니다.

상관관계 vs 인과관계

상관(correlation)은 "두 값이 함께 움직이는 경향"이고, 인과(causation)는 "하나가 다른 하나의 원인"입니다. 둘은 전혀 다릅니다. 데이터 과학에서 가장 흔한 실수가 이 둘을 혼동하는 것입니다.

함께 늘어나지만(양의 상관) 서로의 원인은 아님 — 진짜 원인은 "여름 더위"
🚫 상관 ≠ 인과

아이스크림 판매와 익사 사고는 함께 늘지만, 아이스크림이 익사를 일으키진 않습니다. 둘 다 여름 더위라는 숨은 변수(교란변수)의 결과입니다. 인과를 주장하려면 보통 통제된 실험(A/B 테스트)이 필요합니다.

데이터 시각화 원칙

좋은 그래프는 한 번에 핵심을 전달합니다. 시각화의 목적은 예쁘게 꾸미는 게 아니라 정직하고 빠른 이해입니다.

  • 목적에 맞는 차트 — 추세는 선그래프, 비교는 막대, 분포는 히스토그램, 관계는 산점도.
  • 축은 0부터 — 막대그래프에서 축을 잘라내면 차이를 과장하게 됩니다.
  • 잉크를 아껴라 — 불필요한 격자·3D·장식(차트정크)을 줄이세요.
보여주려는 것적절한 차트
시간에 따른 변화선그래프(line)
항목 간 크기 비교막대그래프(bar)
값의 분포히스토그램·박스플롯
두 변수의 관계산점도(scatter)

plt.xlabel("점수"); plt.ylabel("빈도")
plt.show()

# 두 변수 관계 — 산점도

plt.show()

특성 공학 (Feature Engineering)

특성(feature)이란 모델이 입력으로 쓰는 각 열(변수)입니다. 특성 공학은 원본 데이터를 가공해 더 쓸모 있는 입력을 만드는 작업입니다. 좋은 특성 하나가 복잡한 모델보다 성능을 크게 끌어올리기도 합니다.

  • 인코딩 — 범주형("서울","부산")을 숫자로 바꾸기(원-핫 인코딩).
  • 스케일링 — 단위가 다른 값(키 cm, 몸무게 kg)을 같은 범위로 맞추기.
  • 파생 변수 — 생년월일에서 "나이", 날짜에서 "요일"을 뽑아내기.
features.py
# 범주형 → 숫자 열들로(원-핫 인코딩)
df = pd.get_dummies(df, columns=["city"])

# 파생 변수: 생년에서 나이 계산
df["age"] = 2026 - df["birth_year"]

# 스케일링: 0~1 범위로 정규화
col = df["income"]
df["income_norm"] = (col - col.min()) / (col.max() - col.min())

빅데이터와 분산처리

데이터가 한 컴퓨터의 메모리·디스크로 감당 못 할 만큼 커지면 빅데이터(big data)의 영역입니다. 흔히 3V로 설명합니다 — 양(Volume), 속도(Velocity), 다양성(Variety). 해법은 한 대를 키우는 게(수직 확장) 아니라 여러 대에 나눠 처리하는 것(수평 확장, 분산처리)입니다.

마스터가 일을 쪼개 여러 노드에 분배 — 평범한 컴퓨터 수십~수천 대로 처리
📌 수직 확장 vs 수평 확장

한 대를 더 강하게(수직)는 한계가 있고 비쌉니다. 평범한 컴퓨터를 여러 대 묶는(수평) 방식이 빅데이터의 표준입니다. 하둡(Hadoop)·스파크(Spark)가 대표 도구입니다.

맵리듀스 (MapReduce)

맵리듀스는 큰 작업을 여러 노드에 나눠 처리하는 대표적인 분산 처리 모델입니다. 이름 그대로 두 단계입니다 — Map(각 조각을 (키, 값) 쌍으로 변환), Reduce(같은 키끼리 모아 합치기). 그 사이 같은 키를 한곳에 모으는 셔플(shuffle) 단계가 있습니다.

단어 세기 예: Map으로 (단어,1) 만들고 → 같은 단어 모아 → Reduce로 합산
wordcount.py
# 맵리듀스 개념을 단순화한 단어 세기

def map_phase(line):
    # 각 줄을 (단어, 1) 쌍들로 변환
    return [(word, 1) for word in line.split()]

def reduce_phase(word, counts):
    # 같은 단어로 모인 1들을 합산
    return (word, sum(counts))

# Map → Shuffle(같은 키 묶기) → Reduce 순으로 동작

데이터 파이프라인

데이터 파이프라인은 데이터가 출처에서 분석 결과까지 흐르는 자동화된 통로입니다. 흔히 ETL(Extract 추출 → Transform 변환 → Load 적재)로 부릅니다. 매번 손으로 처리하는 대신, 정해진 순서대로 자동 실행되게 만듭니다.

ETL: 추출 → 변환 → 적재. 매일 자동으로 흐르도록 스케줄링
💡 재현 가능성

분석은 같은 입력이면 같은 결과가 나오도록 코드로 남기세요. 손으로 엑셀을 만지면 6개월 뒤 누구도 그 숫자를 재현하지 못합니다.

윤리와 편향

데이터는 중립적이지 않습니다. 과거의 편향이 데이터에 그대로 담겨, 분석과 모델이 차별을 학습하고 증폭할 수 있습니다. 채용·대출·치안 같은 영역에서 이는 실제 사람에게 피해를 줍니다.

  • 표본 편향 — 일부 집단만 과/소대표되면 결론이 왜곡됩니다.
  • 개인정보 — 식별 가능한 데이터는 익명화·최소 수집 원칙을 지켜야 합니다.
  • 설명 가능성 — 왜 그런 결정이 나왔는지 사람에게 설명할 수 있어야 합니다.
🚫 데이터는 거울이다

편향된 과거 데이터로 학습하면 모델은 그 편향을 "정답"으로 받아들입니다. 데이터 출처와 수집 방식을 의심하는 것이 분석가의 첫 번째 책임입니다.

🎯 핵심 정리
  • 데이터 과학은 수집 → 정제 → 탐색 → 시각화 → 결정으로 흐른다.
  • 현실 데이터는 더럽다 — 정제가 분석 품질을 좌우한다(garbage in, garbage out).
  • EDA는 가설보다 먼저, 데이터가 스스로 말하게 두는 단계다.
  • 평균은 이상치에 약하고, 한쪽으로 치우친 데이터엔 중앙값이 낫다.
  • 상관은 인과가 아니다 — 숨은 변수를 항상 의심하라.
  • 빅데이터는 한 대를 키우지 않고 여러 대에 나눠 처리한다(맵리듀스).
  • 데이터는 과거의 편향을 담는다 — 윤리와 편향을 항상 점검하라.
결측치(Missing Value)
값이 비어 있는 칸. 삭제하거나 대표값으로 채워(대치) 처리한다.
이상치(Outlier)
정상 범위를 크게 벗어난 값. 오류일 수도, 진짜 특이 사례일 수도 있다.
EDA
탐색적 데이터 분석. 통계 요약과 그래프로 데이터의 생김새를 파악하는 단계.
분산·표준편차
값들이 평균에서 얼마나 흩어져 있는지를 나타내는 척도.
상관 vs 인과
함께 움직이는 경향(상관)과 원인-결과 관계(인과)는 다르다.
특성 공학
원본 데이터를 가공해 모델에 더 유용한 입력(특성)을 만드는 작업.
맵리듀스
큰 작업을 Map(변환)·Reduce(합치기)로 나눠 여러 노드에서 처리하는 분산 모델.
ETL
추출·변환·적재. 데이터를 출처에서 분석 저장소까지 옮기는 파이프라인 단계.
편향(Bias)
데이터·분석에 담긴 치우침. 차별과 왜곡된 결론의 원인이 된다.
